引言

直播电商的蓬勃发展已重塑了中国的电商竞争格局，并预计在2023年达到近2.5万亿的市场规模，这种模式也为美团带来了新的增长机会。今年Q3优选启动了直播项目，本文探讨了优选对直播机会的判断，以及在定位和选型上的思考，并介绍了如何运用AI技术和司内的数字人能力解决业务高运营成本的问题。

### 一、美团优选做直播的机会

直播电商的发展催生了中国电商新的商业模式，更重塑了电商竞争格局。中国直播电商起源于2016年，之后大致经历2017-2018年的快速拓展期、2019-2020年的百花齐放期以及2021至今的全民直播三大阶段。2023年直播电商的市场规模接近2.5万亿，全年增速预计在35%~40%。直播电商的规模和增长速度已经充分证明了其对内容和电商平台在用户规模和交易规模方面的巨大价值。对于优选而言，我们对直播电商的特点进行了分析，结合业务的实际情况，判断直播在提升转化效率和获取流量方面都具有很大的机会。

一方面，相较于传统货架，直播可以提供高密度的信息传递和更实时的售前客服，对于重决策和非标的商品，能够极大地提高用户决策效率。优选的核心品类生鲜，具有极度非标的特性，即使是同一品种的苹果，不同批次、不同产地之间，用户的观感、口感也可能是不相同的。通过直播，用户可以清晰、直观地看到商品的规格和质量，弥补了线上渠道天然存在的信息不对称。其次，直播中实时互动的卖货形式更具感染力，能够激发用户潜在需求，结合美团平台对直播内容的流量扶持和优选在团长微信群传播来撬动视频号公域流量，直播内容可以在消费者产生购买意图之前更早触达用户，提前锁定需求，从而获得流量和购买增量。

### 二、美团优选直播定位和选型

对标头部的直播电商平台，无论抖音、快手、还是淘宝直播，业务第一阶段都是建立独特心智以完成市场区隔。抖音和快手作为内容平台，以内容变现为核心定位。抖音为重新定义电商的全域兴趣电商、快手是重构消费决策的信任电商。淘宝是电商平台，定位为电商内容化，通过专业内容带动销售转化。优选作为电商平台，用户的第一心智为“购物”，因此**定位**更适合对标淘宝，即**电商内容化**。通过更鲜活的商品展示形式、专业的互动讲解和及时的售前客服，提高用户需求满足效率，进而在供给侧撬动更低的价格，带动更多的流量和交易。
![[Pasted image 20260420174319.png|514]]
其次从**选型**上来看，业界通常有3种模式：官方直播、店铺直播和达人直播。在内容电商领域，抖音和快手初期主要依靠达人直播来实现内容变现，随后为了发展货架电商鼓励商家自播，长期来看是达人直播和店铺直播并重。淘宝直播发展早期依靠达人吸引外部流量，但头部主播马太效应突出，长期来看平台战略重心是以商家自播为主，达人直播为辅。美团外卖、到店今年Q2开始试水直播，初期通过官方直播保障体验建设直播心智，长期来看鼓励商家直播丰富生态内容，让直播成为商家新的营销工具是必然的趋势。

优选和平台电商不同，有限货架的模式下商品数量有限，每个商家所售卖的商品数少，且没有店铺私域场，因此店铺直播效率差，难形成规模。所以优选更适合官方直播和达人直播。考虑到当前直播业务是发展初期，官方在内容质量、运营和财务成本、测试速度上都有明显的优势，综上我们认为官方直播是现阶段更合适的选择，暂不考虑达人直播。

| 维度   | 官方直播                       | 店铺直播                       | 达人直播                                |
| ---- | -------------------------- | -------------------------- | ----------------------------------- |
| 特点   | 平台做官方直播场,售卖平台商品,通过公域聚流量做认知 | 商家直播售卖商品,通过私域聚流量或公域买量      | 达人分销不同商家商品,通过私域或公域聚流量               |
| 主播目的 | 聚流量、冲销量、提转化                | 品牌宣传、提转化、冲销量               | 分销赚钱                                |
| 典型业务 | 美团外卖、美团到店                  | 淘宝、拼多多、抖音                  | 抖音、快手、小红书、淘宝                        |
| 优选选型 | 适合:建立用户认知;提供额外流量池,可做低价心智   | 不适合:商家商品少无私域,爆款引流不成立,分发效率低 | 现阶段不适合:无达人体系,培育成本高;初期流量少,引入达人增加财务成本 |
确定选型为官方直播后，接下来要解决的**难点是直播运营成本高的问题**。由于优选的区域化经营特点，全国各个网店出售的商品（SKU）大多不同（尤其是生鲜品），即便商品相同，也会因区域运营策略、供应商不同从而导致价格不同，因此单场直播需要围绕网店的粒度展开。此外，优选全国的网店数量庞大，直播环节多，流程复杂，直播所需要的内容策划和前台销售能力需要专业积累，各区域的运营能力参差不齐，因此**单场直播总计要耗费数千人时**，运营成本非常高的同时也很难保证效果。

在这样的背景下，我们梳理了直播的运营流程，并调研了司内外的解决方案。目前业界已经有比较成熟的智能脚本生成、智能客服和数字人直播的方案。在今年大模型技术的加持下，内容生成的效果和数字人能力也得到了大幅提升。因此我们在Q3启动了AI直播项目，通过大模型+自研算法选品+大模型生成直播内容+对接视觉平台提供的数字人服务+自动创建直播间等能力，完成了AI直播流程的跑通，并计划在近期上线7*24小时全区域自动化直播能力。能力上线后，对比传统的人工直播，运营人效提升20倍左右。未来我们会在直播脚本内容的丰富度、商品呈现、数字人表达的拟人化、数字人互动、直播转化体验、营销玩法、智能客服等层面持续投入。探索日常AI数字人直播、重要节点AI+人播的运营模式，不断地提升直播为业务带来的增量收益，降低运营成本

**4.3、优选AI直播关键能力介绍**

**1、智能选品与大模型生成直播文案**

在AI选品方面，我们采用了大模型与规则相结合的策略。大模型用于召回一系列可能的选品结果，然后通过规则筛选出适合的品类。对于直播文案生成，由于其复杂性，我们在项目的不同阶段采用了不同的策略。

（1）冷启动阶段

在项目初期，我们面临着文案数据稀缺的问题。为解决这一问题，我们选择使用大型语言模型生成直播文案，以利用其强大的生成能力和广泛的通用知识积累文案数据。然而，这种方法也面临着一些挑战：

- 文案过长的问题：由于直播文案的长度较长，模型在生成时可能难以兼顾内容的丰富度和表达的流畅性。
    

- 领域知识缺乏的问题：大型语言模型的训练语料可能无法覆盖特定领域的数据，导致模型在生成直播文案时无法准确、具体地描述文案内容。
    

为解决这些问题，我们采用了以下两种策略：

- 分段生成策略：我们将长文案拆分为八个分段，每个分段只描述文案的特定部分，以避免生成过长的文本降低内容的可控性和文本的流畅性。实践证明，这种方法不仅可以保证文案内容的质量，还有助于提高人工审核的效率。
    

- 检索增强策略：在缺乏领域知识时，大型语言模型可能会生成空泛无意义甚至似是而非的内容，影响文案的质量。为了增加文案的信息量，我们采用了检索增强的方法。例如，对于描述商品“使用方法”的分段，我们会检索商品百科来召回可能的菜谱，使得这个分段的内容更加具体、富有信息量。
    

然而，使用大型语言模型生成文案的成本较高，效率较低，无法满足项目扩展阶段的需求。因此，我们在扩展阶段采用了新的策略。

（2）扩量阶段

在项目的扩量阶段，我们利用冷启动阶段积累的文案数据，选择基于开源的7B大语言模型训练垂直领域的模型，以降低调用成本，提高生成效率，满足大批量请求的需求。具体方案主要包括监督微调（Supervised Fine-Tuning, SFT）和人类反馈对齐（Reinforcement Learning from Human Feedback, RLHF）两种技术。

- 监督微调：具体的方法是指令精调，我们利用冷启动阶段收集的数据做监督微调，使较小的7B模型学习到文案生成任务上的指令遵循偏好，从而以较小的参数量得到和大语言模型相近的输出。
    
- 人类反馈对齐：我们还在指令精调的基础上，利用人类反馈改善模型的生成偏好。例如过滤一些有“极限词”倾向的描述。

**2、AI直播文案的审核与训练**

每一个商品的文案是由多个类型的段落拼接而成，一个完整的文案示例如下表。

| 段落类型 | 段落内容 |
|---------|---------|
| 基本信息 | 下面介绍的是安心购优选特级去皮少脂大排里脊肉哟,每盒的重量是450克,误差大约是30克。 |
| 核心卖点 | 这款猪里脊肉呢,去皮少脂,吃起来更健康,更放心哈。 |
| 使用场景 | 它是你做家常菜的好帮手,无论是炒菜、炖菜还是烧菜,都能让你的菜品变得更美味。特别是在聚会的时候,用它来做菜,绝对能让你的朋友们赞不绝口哟。 |
| 使用方法 | 你可以用它来做青椒炒里脊,也可以做雪菜炒冬笋,都是非常美味的家常菜哈。 |
| 促销利益点 | 今天有个好消息要告诉大家,我们的商品今日有秒杀特价,比平时更实惠哟。 |
| 口碑信息 | 而且这款商品已经有3.9万的小伙伴下单了,近期销量已经突破3万多件,还在热销榜上拿下了第5名,已经有5万多的小伙伴给出了好评呢。 |
| 售后政策 | 我们的售后政策也很给力,72小时内坏品我们都包赔,退款也是闪电速度。赶快来抢购吧! |

AI直播的发展初期，需要运营进行文案审核。因此我们搭建了文案审核后台，运营审核的颗粒度是段落，后台维护了每个段落的审核状态机。在运营审核完毕后，会将通过的段落组装成最终文案，推送给画眉AI创作平台生成视频。对于不通过的段落以及运营编辑的驳回理由定时回传给算法侧，供算法模型进行学习。（需要注意的是，基本信息段落不通过时，整段文案将无法通过）

算法侧定时处理回传数据、将其转换成人类标注的数据。在RLHF训练过程中，这些数据被用于训练垂直领域的模型，目的是对齐人类的偏好，使训好的模型生成的文案能够更好地符合实际的需求。随着算法模型的不断优化，运营审核的通过率也会随之提高，在通过率到达一定阈值后（95%），在保时洁审核的辅助下，运营只需要抽检即可。在未来，文案审核的预期终态是使用GPT审核和保时洁审核代替运营审核，将这个耗费人力的过程实现完全自动化。

**3、数字人直播视频生成（文案 -> 视频）**

当准备好自动选品与文案生成后，接下来的问题是如何生成数字人的视频。这里我们与我司的画眉AI创作平台进行了深入合作，使得优选数字人直播项目顺利开播。

在优选AI直播系统中，我们使用了画眉AI创作平台（以下简称画眉）的以下能力：

- 在文案通过审核后，优选将文案发送给画眉，画眉生成离线数字人视频后将视频的URL信息进行广播，优选监听到消息后进行储存并下发到客户端，可以实现单个商品的回放能力
    
- 在直播开始前，优选再次将文案发送给画眉，画眉进行直播间初始化并进行推流准备
    
- 直播开始时刻，优选会通过接口触发画眉进行推流，此时客户端通过拉流后便可以看到数字人直播
    
- 在直播过程中，当需要和进入直播间或者下单的用户进行互动时，优选会发送实时计算好的互动文案到画眉，供画眉插入到直播流中
    

**4、数字人讲解的商品置顶**

在传统直播中，如果主播想将当前讲解的商品进行置顶，需要在主播端去操作该商品置顶，这给主播增加了不少的操作成本。而在AI直播中，显然再安排专人去操作置顶会带来比较大的人工成本，我们的期望是能够实现商品随着讲解的进度自动置顶，以及自动出现商卡弹窗。

![[Pasted image 20260420175748.png]]
在设计商卡自动弹出初期，有两种解决思路：① 客户端通过SEI编解码识别出视频流里面的信息； ② 服务端通过画眉AI创作平台推流的广播消息来感知推流进度。出于开发成本等因素的考虑，我们选择了后者。

在AI直播中，一个商品对应着一段视频，通过监听并解析画眉AI创作平台在每一段视频推流之前发送的广播消息，服务端就可以感知到商品切换并解析出具体的商品id。

接下来需要把切换的动作通知到前端，这里复用了美团直播平台的pike消息，通过长连接的方式通知到前端，前端接收到消息后从而可以进行相应的动作，即商品置顶和展示商品弹窗。

在这个过程中，为了防止当初推送给画眉AI创作平台消息中的商品售罄，在发送消息之前会再请求一次算法侧进行同品替换，售罄的品会被替换为可以复用文案内容的同品。同品替换原理是通过融合了末级类目、关键属性、规格等商品信息的CSPUID进行检索，在检测到商品售罄时，算法侧基于CSPUID对网店下在售商品进行召回排序，返回最合适的同品。

需要注意的是，AI直播推流采用的是RTMP协议，拉流是FLV，在画眉AI创作平台推流和客户端拉流的过程中，会存在一定的延迟。为此消息也需要做成延迟的，对商品切换和视频的切换时机进行适配，从而达到最佳效果。

**5、AI主播实时互动**

目前AI直播间在做的互动包括两种：① 对进入的用户打招呼； ② 对刚下单的用户进行感谢。这两者的用户信息可以分别通过记录访问直播间接口的用户和监听优选交易侧的提单广播消息获取。当一段商品讲完后，会对快结束时进入的用户和下单用户进行打招呼和感谢，这里的难点在于，如何感知到这个商品快要讲解结束了。目前的解决方案是，将文案按照需要感知行为的时机进行拆分，这样能感知到其中每一段的推流时机。

举个例子，我们现在要对一个商品介绍进度在50%-80%之间进入直播间的最后3个用户打招呼（设置80%是因为在商品结束后插入实时互动的内容需要提前推送，有10s左右的准备时间）。目前的实现方案是，将整个文案按逗号或句号拆分为n句，则将文案拆分为0~0.5n，0.5n~0.8n，0.8n~1n三段，并将这三段发送到画眉AI创作平台生成三段视频，这样便能感知到第二、三段文案的推流时机了，从而记录相应时间内有相关行为的用户。

在实际使用中，打招呼和下单的计算时机是可配置的，在每一个文案按照配置进行剪裁后，最后加上欢迎语、串场话术、结束语拼接好后发送给画眉AI创作平台。

**6、正在优化中的能力**

目前AI直播系统已经初具规模，为了让AI直播向着智能化、自动化不断发展，当前还有很多工作在落地中：

- 批量创建AI直播间并同时开播的能力：基于优选的业态特性，每个网店都需会单独进行选品并生成数字人视频，使用虚拟账号进行同时开播，进一步扩大规模
    
- 自动控制开关播：目前AI直播虽然推流已经实现自动化，但是开关播还需要运营在直播平台进行操作，实现自动开关播后可以进一步减少人工成本
    
- 文案审核自动化：目前随着算法模型的不断学习，运营的审核成本也在逐渐减少，未来的终态是用机审完全代替人审
    
- 进一步增强互动能力：除了目前支持的打招呼和感谢下单，未来还可以考虑评论自动回复等，让直播效果进一步接近真人直播
