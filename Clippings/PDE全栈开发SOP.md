## 前后端开发

#### 一、技术方案设计

建议以下两个部分并行推动，因为后端逻辑会制约产品需求和前端demo的实现，而前端页面demo可以帮助我们和需求方快速确认产品的细节

1. 前端demo产出：将需求澄清环节沉淀的需求文档喂给catpaw，让它遵循其他页面的样式，快速帮你搭建产品的demo样态，可以让它mock一批数据这样后端api就不会是前置依赖了，注意让他单独建个mockdata不要把mock数据分散到代码各处，另外mock数据尽可能和上线后的真实数据一致，这也是在让模型更好理解你的需求。
    
2. 后端技术方案设计：
    
    1. 已有api梳理：根据现在产品已有功能和catpaw对于当前代码库的讲解，梳理目前有哪些api是可以复用的，不要重复造轮子
        
    2. 数据底表设计：暂时忽略，已经帮大家建好，说明如下
       **checkin（打卡记录表）**
       **说明：** 存储用户的打卡记录。同一用户同一地点同一天只能打卡一次（唯一约束）。
       **字段说明**![[Pasted image 20260429153042.png]]
    3. PRD产出：建议完成前面两点后确定一版最终版的PRD，这版需要包括所有需求的具体实现，具体页面的样式。这个文档主要有两个目的：（1）供后续可能需要快速了解该项目的同学阅读；（2）后续可以给喂给ai快速产出测试文档，AI coding中测试验收是远比写代码重要得多的事情。
#### 二、前端开发

1. 前端打磨：完成前期技术方案设计后就可以介入真正的开发了，前端其实大部分代码在demo产出阶段其实就已经完成了，这一阶段更多的是细节的打磨和对齐，因为AI coding的产物往往在细节上会出现瑕疵。
    
2. 代码审核：在写完前端代码后，需要对代码进行review，建议至少进行三次，每次需要新开会话，最好选用分别选用御三家进行（Claude/gpt/Gemini），根据暴露的问题进行修复。
    
3. api梳理：审核完成的前端代码，就可以让ai给你梳理需要的后端api文档了，让它给一份尽可能详细的文档，方便后端开发。
    

#### 三、后端开发

1. 规划文档：在catpaw中打开后端项目，然后把前端项目以及上一节完成的api文档的地址复制给它，最后在加上方案设计环节产出的数据底表设计文档（学城文档链接），让catpaw帮你生成一份后端开发规划文档。
    
2. 代码开发：接下来就是根据规划文档让它按部就班地进行开发即可，注意如果可以不要改动非本次开发的代码，需要跟它强调这件事，并且在accept all前关注一下改动的文件名称。
    
3. 前端确认：完成开发后，在前端项目中复制给catpaw后端项目的地址，让它确认是否提供的api是满足需求的。
    
4. 代码审核：在写完后端代码后，需要对代码进行review，建议至少进行三次，每次需要新开会话，最好选用分别选用御三家进行（Claude/gpt/Gemini），根据暴露的问题进行修复。并且让catpaw帮你估算一下当前的性能瓶颈，是否满足你的需求，不满足则需要进行一轮额外的优化。
## 前后端联调

在完成前后端代码的开发后，我们就需要前后端的测试环境联调了，这个阶段是坑最多也是考验人的阶段。

#### 一、后端部署

由于前后端都是我们自己做的，在部署环节我们可以暂时只部署后端代码，前端本地进行联调即可，这样修改起来会更加方便，后端部署主要分为这几步：

1. 找baixiaoxiang帮加服务（com.sankuai.ulivepde.runmap）权限，进入[https://dev.sankuai.com/cargo/stack](https://dev.sankuai.com/cargo/stack)，然后按如下操作
    
    1. 新建编排
       ![[Pasted image 20260429153120.png]]
    2. 选择a步骤中添加权限的服务：com.sankuai.ulivepde.runmap
       ![[Pasted image 20260429153130.png]]
2. 在后端项目中，跟catpaw讲让它帮你提交推送你的代码到你的分支（不要提交到master）
3. ​​点击构建，选择你的分支，点击构建镜像并部署
   ![[Pasted image 20260429153205.png]]
4. 如果一切顺利，会收到部署成功的消息，并且环境信息中的机器列表会显示running：
   ![[Pasted image 20260429153216.png]]
   ![[Pasted image 20260429153228.png]]
5. - 但很遗憾大多数时候并不会这么顺利，报错失败才是常态，所以需要我们掌握基础的调试方法，常用的有两种：
    1. 日志分析：直接复制粘贴部署时报错的日志给到claude code
        
    2. 后端排查：如果单纯的报错日志没有办法解决，则需要使用跳板机进行后端排查：[https://jumper.mws.sankuai.com/](https://jumper.mws.sankuai.com/)，登录之后需要连到你的主机上
       ![[Pasted image 20260429153259.png]]
       然后让claude code给你用于后端排查的命令，复制粘贴运行后，再把结果复制给claude code即可。
       ![[Pasted image 20260429153313.png]]
#### 二、联调

1. 成功部署之后我们就可以开始联调了，注意需要切换到你的泳道上，不了解如何切换泳道的同学可以参考这个：[ModHeader](https://km.sankuai.com/page/869134009)
2. 冒烟测试：把前期产出的PRD喂给claude code，让它给到一个冒烟测试的流程，跑通产品逻辑的主流程
3. 细节确认：跑通产品主流程后再测试确认功能细节是否都符合预期
4. 大多数时候联调阶段会遇到非常多的问题，因此需要我们掌握基本的排查方法：
    1. 右键点击检查，查看控制台中的日志，把报错信息复制粘贴给catpaw，让它排查/查看网络中请求的响应，把你对应操作的响应复制给catpaw
       ![[Pasted image 20260429153430.png]]![[Pasted image 20260429153441.png]]
    2. 如果报错信息/响应中没有实际信息，就让catpaw修改后端代码把补充更完善的报错信息，辅助我们排查
    3. 联调中如果涉及修改后端的代码，则需要修改后commit push到你的分支去，然后重新进行建构部署
5. 在本地联调完成后，把前端代码在talos中部署到相同泳道的测试环境中，然后再进行一轮走查验收，确保测试环境前端页面及接口调用正常
