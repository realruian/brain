# 
一. 背景

依托于移动通信技术的蓬勃发展，网络带宽的进一步提升，用户收看视频的成本更低；同时受益于手机算力的提升、云计算与智能硬件门槛降低，视频制作成本进一步降低。在此基础上，短视频媒体提供了相较于传统图文内容体验更佳的内容获取范式，成为了新的头部媒体形态，用户也越来越习惯消费视频创意。伴随短视频市场的火爆，媒体方以及代理方也纷纷布局短视频的信息流业务，视频信息流广告的售卖呈现增长趋势；广告主对视频信息流广告的认可度、重视度也在提升，从测试、试验阶段走向常态化的投放。

基于广告创意的行业发展趋势“__原生化、视频化、动态化”__+本地生活平台的小微商户在线营销和创意制作能力弱的业务特色，需要我们提供更简单的视频制作工具来协助完成视频创意制作。同时考虑到目前的主流广告投放模式为暗投，基于暗投的模式，整个业务侧对创意基本无明确图文或创意诉求，核心诉求为广告效果和转化率，到店广告平台需要为中小商户批量制作视频供给投放。出于广告主特征和业务特点考虑，我们需要供给可程序化批量为广告主生产视频的能力，同时需要在视频的制作中体现对各个行业的理解与洞察，所以需要建设富媒体视频可视化搭建及配套生成的能力。

# 
二. 技术难点与挑战

## 
可视化模板制作

视频创意模板的可视化制作目前面临三个挑战：

- 

如何进行工具拆分，拆分逻辑是什么？

- 

如何选择动效方案，可以尽可能降低技术实现成本，同时发挥设计能力与优势？

- 

如何帮助 UED 以 no-code 的方式自主生产，功能性和交互性怎样平衡？

## 
程序化视频生成

程序化视频生成面临两大挑战：

- 

如何从服务端还原视频模板设计器中丰富的视频设计效果？

- 

如何实现批量化视频生产，实现快速批量大规模的视频上线投放？

### 
动效方案

视频创意的动效方案有三个方向：

- 

图像处理，即对图像进行像素级处理的能力，对应产品功能为风格滤镜、3D转场、视频抠图、AI换脸等；

- 

叠加装饰物，如叠加遮罩蒙层 / 装饰动画 / 文字等，对应功能为动态贴图、动态字幕、氛围效果等；

- 

改变图像整体样式，如位移 / 旋转 / 透明度 / 宽高的改变叠加贝塞尔曲线缓动等，对应功能为一般图片效果 、转场；

第一类动效方案强依赖算法，前端以WebGL / canvas实现为主；二三类的实现方式则多种多样，大致有GIF / 帧动画 / CSS / Lottie这几种。

### 
业务流程

drawio{src="https://km.sankuai.com/api/file/cdn/1202264361/1177581607?contentType=2&isNewContent=true" width=1189 height=480.8530161427358 nodeId="892ba895199546f5a649b958f924113e"}:::

### 
面临的挑战

**视频渲染合成能力的不足**

视频创意可视化搭建能力建设中，视频模板编辑器是通过 h5 技术实现的，媒体元素支持 h5 动效、图片、音视频，元素的位置、尺寸、层叠、时间轴的设置上没有任何限制；视频模板渲染合成服务的核心就是将视频模板编辑器前端呈现的内容在服务端还原效果以及合并成一个视频，合成音视频必然涉及使用到视频编辑能力，同时服务侧还需要支持Lottie等复杂特效的合成处理，经调研公司内部已有 Swarm 视频处理公共服务，但编辑能力上暂无法满足，并且缺口较大。

### 
解决方案

为了满足业务节奏，第一阶段业务需求的几种类型的模板需要快速完成迭代，对于支撑视频动效上使用了截帧图的方案，合成上采取了短期先由视觉智能中心团队临时提供 Horus 定制视频编辑接口，用于己支持业务需求的几种类型模板的音视频合成；同时给 Swarm 团队提需求，提供具备复用性的通用接口，逐渐支持匹配富媒体视频可视化搭建的能力。

第一阶段完成的系统仅为了快速支持业务，系统上存在不少问题，比如视频合成步骤定制化程度过高，视频合成的处理步骤只能串行执行，视觉智能中心团队 Horus 服务只是短期提供临时支持，后续需要切换至 Swarm等；考虑到随着 Horus 切换 Swarm，Swarm 能力的逐渐完善等因素，会导致视频模板渲染合成视频的整体步骤产生变化，视频渲染合成服务进行了重构设计，增加了子服务：任务中心，该服务负责协调视频模板制作的各个步骤（子任务），支持并行执行处理各步骤，同时也对视频合成步骤的不确定性进行了解耦。

随着视频渲染合成服务支持了任务中心，以及 Swarm 的视频编辑能力的逐渐完善，并新增支持了视频 + (多)音频跨时间轴合并的能力，视频模板渲染合成服务匹配视频模板编辑器强大而又灵活的模板制作功能的还原（包括支持素材任意配置尺寸、定位、层级、时间轴，支持素材空间位置重叠，时间轴重叠等）变成了现实，视频模板渲染合成服务通过配合 Swarm 逐步完善的视频编辑能力，采取了将"音"与"视"分离，各自还原后，再进行合并的处理方案。

最终视频模板渲染合成服务架构、视频模板渲染合成服务的核心实现方案如下：

### 
服务架构

主要分为三个子服务：

- 

**接口网关**，负责视频模板制作任务各接口的调用层，主要提供的接口包括：发起视频模板制作任务，查看任务制作情况等。

- 

**任务中心**，负责协调视频模板制作的各个步骤（子任务），主要功能包括：解析视频模板，计算出该模板制作的步骤（子任务），分派各步骤（子任务）的执行，处理各步骤（子任务）的执行结果回调等。

- 

**模板渲染**，负责对视频模板视觉部分的内容进行渲染及抽帧处理，主要功能包括：渲染还原视频模板效果，按帧播放，截图处理及上传云存储等。

并对接了 Swarm 视频服务、Horus edfu 特效渲染服务的能力；

### 
核心实现

视频模板渲染合成服务的核心就是将视频模板编辑器前端呈现的内容在服务端还原并合并成一个视频，为了匹配视频模板编辑器强大而又灵活的模板制作功能的还原（包括支持素材任意配置尺寸、定位、层级、时间轴，支持素材空间位置重叠，时间轴重叠等），视频模板渲染合成服务采取了将"音"与"视"分离，各自还原后再合并的处理方案。

- 

**分离"音"与"视"**，将模板中视频素材的音频与图像分离。

- 

**还原"音"**，按模板时间轴合并分离出的音频，生成还原的音频。

- 

**还原"视"**，按模板时间轴将所有"视"的内容进行渲染合并，生成还原的视频。

- 

**合并"音"与"视"**，合并还原的音频与还原的视频，生成最终还原的视频。

各子服务的核心实现如下：

**（1）接口网关**

- 

**发起制作任务，**该接口会接收业务方的视频模板信息，生成一个视频模板制作任务，并发送至任务中心服务执行。

- 

**查询制作任务，**该接口会根据业务方提供的视频模板制作任务 id，查询到该制作任务的当前执行的状态，步骤的详情等信息。

**（2）任务中心**

- 

**模板解析与步骤生成，**在接到视频模板制作任务后，会根据约定的 DSL 规则对模板进行解析，根据解析的结果生成制作步骤、执行顺序及依赖关系，制作步骤包括动效截图、帧图合并视频、音视频合并、视频拼接、视频贴图 、视频叠放、音频拼接、音视频切割、特效附加等。

- 

**步骤分派与步骤回调，**在制作步骤生成后，依照步骤的类型与执行顺序会将最初的步骤发送至对应的服务执行，例如动效截图步骤会发送至模板渲染服务，帧图合并视频步骤会发送至 Swarm 视频服务，特效附加步骤会发送至 Horus edfu 特效渲染服务；各步骤执行结束后会发送回调给任务中心，在步骤执行后的回调中，根据步骤的依赖关系决定需要发送的下一个步骤，直到最后一个步骤执行结束。

**（3）模板渲染**

- 

**渲染还原与按帧播放，**在动效截图的步骤后，会通过 headless chrome 的方式将模板渲染还原，通过定制的模板渲染器将每一帧单独渲染出来，这一部分正在持续优化向更高性能的绘制方案转变。

- 

**按帧截图与上传，**在模板的每一帧渲染出来后，对模板区域进行截图，并上传至 S3 保存，供后续的步骤使用。

# 
五. 未来规划

广告创意视频模板的可视化搭建能力建设目前仍在保持高速迭代中，目前在工具能力、合成性能上仍有提升空间。

- 

**工具能力方面**，目前已达成“赋能UED”的阶段性目标，下阶段目标将是“建设像素级的图像编程能力”，基于Canvas与WebGL渲染，提供诸如3D转场、风格滤镜、视频抠图、图像特效等产品功能；

- 

**合成性能方面**，从无头浏览器截图转向更高性能的绘图方案，通过程序化的方式进一步提高服务的吞吐量，持续与音视频转码服务（Swarm）团队沟通及提需，进一步完善视频编辑能力，简化合成步骤，提升合成效率；

- 

**合成质量方面**，会考虑接入视觉智能中心团队提供的质量评分的能力，完善创意的质量评价体系；
