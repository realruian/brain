## 基础UI规范

以下规范在所有场景的 Step 4 中生成或修改代码时**必须**遵循。

---

### 一、组件使用规范

#### 1. Tabs 组件样式处理

`Tabs`（`@ss/mtd-react3`）默认无上边距，无论是否有包裹容器，**必须**在 `Tabs` 组件上设置 `paddingTop: 12`。

```jsx
<Tabs style={{ paddingTop: 12 }}>...</Tabs>
```

**mtd-prime-tabs-nav 无内容时，必须隐藏内容区占位元素**，否则与下方容器产生多余间距：

- React：`.mtd-prime-tabs-content { display: none; }`
- Vue：`.mtd-tabs-content { display: none; }`

#### 2. 禁止使用 Emoji

**严禁**在页面中使用 Emoji 表情（如 🎉 ✅ ❌ 📋 等）作为 UI 元素。如需图标，应使用 MTD 的 `Icon` 组件替代。
图标名称及使用方法见 `doc/icon_name.md`。若需要的图标不在列表中，选择语义最接近的替代图标，绝对不能用 emoji 代替。

```jsx
// ✅ 正确
<Icon type="check-circle-o" />

// ❌ 错误
<span>✅ 操作成功</span>
```

#### 3. Icon 组件风格统一

`Icon` 组件分为**线型**和**面型**两种风格：
- **线型**：图标名以 `-o` 结尾，如 `check-circle-o`、`close-circle-o`
- **面型**：图标名不带 `-o` 后缀，如 `check-circle`、`close-circle`

**同一区域内的 Icon 必须统一风格，不得线型与面型混用。**

```jsx
// ✅ 正确（统一使用线型）
<Icon type="check-circle-o" />
<Icon type="close-circle-o" />
<Icon type="info-circle-o" />

// ❌ 错误（线型与面型混用）
<Icon type="check-circle-o" />
<Icon type="close-circle" />
```

### 三、视觉美学指南

在遵守 MTD 组件规范和 Token 体系的前提下，追求有辨识度、有质感的页面设计。以下原则帮助你超越"能用"，做到"好看"。

**阴影使用约束（强制）**
默认采用“低阴影/无阴影”策略，**尽量避免使用阴影**。
页面层级区分优先使用：背景色对比、边框、间距、分区留白，不优先依赖 `box-shadow`。
仅在**确有上下层级或浮层语义**时使用阴影（如 `Modal`、`Drawer`、`Dropdown`、悬浮操作条、需要强调可点击的浮起卡片）。
即使使用阴影，也优先 `--shadow-1`，避免大面积或高强度阴影（`--shadow-3/4` 仅限少数关键浮层）。

**空间构图**
留白是设计的一部分，不是浪费空间。**模块内的上/下/左/右间距必须尽量统一，禁止同一模块出现无规则的间距跳变**。卡片内边距统一用 `24px`，相关元素间距统一用 `8px` 或 `16px`，不同功能区块间距统一用 `24px`；如需破例，必须有明确语义（如主次层级、重点强调、响应式折行）。避免把所有内容塞满屏幕——适当留白让页面呼吸，也让重点更突出。统计卡片、操作按钮组、筛选区域之间的间距需保持同一节奏，体现秩序感。

**避免平庸设计**  
不要让每个页面都长得一样。同样是列表页，可以在筛选区的布局、操作列的交互方式、空状态的设计上做出差异。数据看板可以用不对称的卡片网格而非整齐的等宽列。表单页可以用分步骤的 Steps 引导，而不是一个超长的滚动表单。在 MTD 组件的约束内，仍然有很大的空间做出有个性的设计。
