## MTD组件库介绍
MTD组件库共有三个npm包，分别为@ss/mtd-react3、@ss/mtd-vue2、@ss/mtd-vue3。其中@ss/mtd-react3是React组件库；@ss/mtd-vue2是Vue2的组件库；@ss/mtd-vue3是Vue3的组件库。注意@ss/mtd-vue2和@ss/mtd-vue3这两个包的组件的源码是一致的，因此组件的知识库也是一致的，只是用户使用的Vue框架版本不同时要用对应的包。

## React组价库和Vue组件库的差异梳理
1. React组件库特有组件：Calendar、DragSelect
2. Vue组件库特有组件：Autocomplete、Sidebar
3. 相似组件：1. 布局组件在React组件库中名字是：Container，在Vue组件库中的名字是：Layout。 2. 下拉菜单组件在React组件库中的名字是：Dropdown，在Vue组件库中的名字是：DropdownMenu

## 组件描述汇总
### Affix
Affix 组件是一个固钉组件，用于在用户滚动页面时将元素固定在视口的特定位置。当页面内容较长且需要确保某些重要元素始终保持可见时使用此组件。
使用场景：固定导航栏、返回顶部按钮、工具栏固定、表单操作栏固定等需要在页面滚动时保持元素可见的场景。

### Anchor
Anchor 组件是一个页面锚点导航工具，用于在长页面中创建快速导航，帮助用户在不同内容区域间快速跳转。
使用场景：文档导航、页内目录、长页面导航、快速定位等需要在页面内快速跳转的场景。

### Announcement
Announcement 组件是一个系统公告展示组件，适用于需要向用户传达重要信息、状态提示或系统通知的场景。
使用场景：系统公告、活动通知、重要提醒、版本更新等需要突出显示信息的场景。

### AutoComplete
AutoComplete 组件是一个带自动完成功能的输入框，适用于需要根据用户输入提供匹配建议的场景。
使用场景：搜索建议、地址输入、代码补全、快速选择等需要智能提示的输入场景。

### Avatar
Avatar 组件是用于代表用户或特定主体的头像组件，支持图片、图标、文字三种类型展示。
使用场景：用户头像、团队标识、项目图标、品牌展示等需要展示图像标识的场景。

### Badge
Badge 组件是一个徽标组件，用于在图标或文字右上角显示红色标记，表示有新内容或待处理的信息。
使用场景：消息提醒、状态标记、数量显示、新功能提示等需要突出显示信息的场景。

### Breadcrumb
Breadcrumb 组件是一个面包屑导航组件，用于显示当前页面的层级、路径、属性等信息，并且能够快速返回之前的任意页面。
使用场景：网站导航、层级路径、当前位置显示、快速返回等需要展示页面层级关系的场景。

### Button
Button 按钮是触发操作的主要交互元素。根据重要性和使用场景，我们提供多种类型、尺寸和状态的按钮。
使用场景：表单提交、操作触发、命令执行、交互响应等需要用户点击操作的场景。

### Calendar
Calendar 组件是一个日历展示组件，按照日历形式展示数据和信息，适用于需要以月视图形式展示和操作日期相关数据的场景。
使用场景：日期选择、日程管理、时间规划、日期展示等需要日期操作的场景。

### Card
Card 组件是一个内容容器组件，适用于将相关信息以卡片形式组织和展示的场景。
使用场景：产品展示、数据卡片、内容分组、信息聚合等需要结构化展示内容的场景。

### Carousel
Carousel 组件是一个轮播组件，适用于在有限空间内循环展示多个平级内容的场景。
使用场景：图片轮播、广告展示、内容切换、产品展示等需要循环展示内容的场景。

### Cascader
Cascader 组件是一个级联选择器，适用于需要从具有层级关系的数据集合中进行分类选择的场景。
使用场景：地区选择、分类选择、层级数据、多级筛选等需要层级选择操作的场景。

### Checkbox
Checkbox 组件是一个复选框，适用于需要从多个选项中选择一个或多个选项的场景。
使用场景：多选项选择、状态切换、批量操作、条件筛选等需要多选操作的场景。

### Collapse
Collapse 组件是一个折叠面板，适用于需要收纳和展示大量内容，通过折叠方式节省页面空间的场景。
使用场景：FAQ展示、内容折叠、信息分组、空间优化等需要折叠展开内容的场景。

### ColorPicker
ColorPicker 组件是一个颜色选择器，适用于需要用户自定义颜色选择的场景，支持多种颜色格式和丰富的交互功能。
使用场景：主题设置、颜色选择、样式配置、设计工具等需要颜色选择的场景。

### ConfigProvider
ConfigProvider 组件是一个全局化配置组件，用于为整个应用或应用的某个部分提供统一的配置，通过 React Context 机制实现全局生效。
使用场景：主题配置、国际化、组件默认值、全局样式等需要统一配置的场景。

### Container
Container 组件是一套页面布局容器组件，适用于需要构建标准页面布局结构的场景。
使用场景：页面布局、内容容器、响应式设计、布局管理等需要容器布局的场景。

### DatePicker
DatePicker 组件是一种用户界面控件，用于选择或输入日期和时间。它提供了一种直观的方式，让用户能够通过交互式的日历界面选择日期，也可以通过输入框直接输入。该组件能够处理单一日期选择和日期范围选择（RangePicker），支持多种日期格式和不同的时间单位选择，包括年份、月份、星期、日期、时间等。
使用场景：日期选择、时间范围、日期区间、日程安排等需要日期选择的场景。

### Divider
Divider 组件是一个分割线组件，用于在界面中创建视觉分隔，帮助用户区分不同的内容区域或功能模块。

## 常见的组件搭配

> ⚠️ 以下规则在 Step 1 识别组件时**必须**优先检查，确保关键布局/导航组件不被遗漏。

---

### 1. 标准后台布局：`Container` + `Menu`

**触发条件**：页面包含「顶部导航栏 + 侧边栏 + 主内容区」等典型后台管理布局。

**必须**将 `Container` 和 `Menu` 加入组件列表并在 Step 2 中查询 API。

**结构说明**：
- `Container`：提供 `Header`、`Aside`、`Main`、`Footer` 四个子组件，负责整体布局框架。
- `Menu mode="horizontal"`：放置在 `Container.Header` 中，实现顶部水平导航。
- `Menu mode="inline"`：放置在 `Container.Aside` 中，实现左侧垂直导航。

```jsx
<Container>
  <Container.Header>
    <Menu mode="horizontal">...</Menu>
  </Container.Header>
  <Container>
    <Container.Aside>
      <Menu mode="inline">...</Menu>
    </Container.Aside>
    <Container.Main>
      {/* 主内容区 */}
    </Container.Main>
  </Container>
</Container>
```

### 2. 顶部水平导航：`Menu mode="horizontal"`

**触发条件**：页面包含顶部水平导航菜单（含 Logo、菜单项、搜索、用户头像等元素）。

**必须**将 `Menu` 加入组件列表，并使用 `mode="horizontal"`。

---

### 3. 左侧侧边导航：`Menu mode="inline"`

**触发条件**：页面包含左侧侧边导航菜单（含菜单项、子菜单等层级结构）。

**必须**将 `Menu` 加入组件列表，放置在 `Container.Aside` 内，使用 `mode="inline"`。

⚠️ **禁止**使用 `Tabs` 作为侧边导航或 Header 筛选交互，即使用户描述中出现了"tab / tabs"等词语，也需判断其真实意图。

---

### 4. 多列栅格布局：`Grid`（`Row` + `Col`）

**触发条件**：页面需要多列并排布局（如多卡片并列、表单多列、统计数字横排等）。

**必须**将 `Grid` 加入组件列表。

**引入方式**（React）：
```jsx
import { Row, Col } from '@ss/mtd-react3';
```

### ⛔ 强制禁止规则（Step 1 中必须逐条核查）

在完成组件列表后，**必须**按以下规则逐条自查，违反任意一条均需立即修正：

**禁止规则 1**：检查框架是否为 React（`@ss/mtd-react3`）——如果是，组件列表中**绝对不允许出现 `Sidebar`**。React 版本没有 `Sidebar` 组件，侧边导航**只能**用 `Container.Aside + Menu mode="inline"` 实现。

**禁止规则 2**：检查页面是否有导航结构——如果有，**绝对不允许**用原生 `<nav>`、`<ul>`、`<li>`、`<div>` 手写导航，**必须**使用 `Menu` 组件实现。

**禁止规则 3**：检查用户描述中是否出现"tab / tabs"——出现时，**不得**直接使用 `Tabs` 组件，**必须**先判断其真实意图：若是页面级导航或侧边筛选，应使用 `Menu`，而非 `Tabs`。
