背景：工程、算法、设计三方共建 Agent 评测体系，意在通过一套体系化、专业化的智能创作评测能力，让 Agent 能够“越用越懂业务”，并通过持续提升评估的准确性，进而实现生图、评估、修正的正向闭环，不断提高采纳率。

### **一、为什么需要**

1. **标准缺失：** 缺乏量化标尺，无法定义“什么是好”，导致行业水位不明。
    
2. **迭代盲目：** 模型更新缺乏对比，无法证明新比旧好，AB 实验效率低。
    
3. **人工瓶颈：** 场景复杂、任务量大，纯靠人工验收无法覆盖全量场景且反馈滞后。
    

### **二、具体做什么**

1. **标准化：** 定义维度（审美+规范），给 AI 设计师下发“考卷”。
    
2. **自动化：** 搭建流水线，实现全程观测、自动评估、异常告警。
    
3. **数据化：** 沉淀线上回流 Case 库，将线上数据反哺模型训练，实现进化闭环。
    

### **三、核心收益（做了能带来什么）**

1. **提效降本：**根据设定的任务集和标准，自动进行验收审核，降低人工成本，提升反馈效率；
    
2. **精准迭代**：通过设置观测指标，量化版本差异，精准核验效果，避免盲目改动；
    
3. **风险可控：**自动化线上巡检 + 告警联动，保障系统稳定；
    
4. **长期进化**：形成 “生图 - 评估 - 反馈 - 迭代” 闭环，Agent 越用越懂业务；
    
5. **树立标杆：**提炼行业级评测标准，对比市面工具，明确竞争身位；
    

### **四、核心目标（要做到什么）**

1. **观测指标体系：**制定可量化的观测指标（包括过程指标、结果指标、效果指标等），涵盖工作过程指标和设计质量指标；
    
2. **自动监测系统：**全程跟踪 AI 工作流程，记录关键数据，发现问题及时告警；
    
3. **数据回流系统：**集中回收问题数据，直接用于 Agent 优化升级；
    
4. **自动评测能力：**结合线上数据和专家评测数据，实现 Agent 自动验收；
    
5. **行业标准输出：**提炼核心指标与数据，打造设计类 Agent 行业评测标准，明确自研工具的行业定位。
