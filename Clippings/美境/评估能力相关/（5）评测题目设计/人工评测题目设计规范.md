# 人工评测题目设计规范

> 本文档规定了如何将评测标准转化为可操作的人工评测题目，适用于所有设计场景的两条评测轨道。

---

## 一、双轨评测体系

| 轨道 | 名称 | 评测对象 | 任务集版本 | 评测方式 | 产出指标 |
| --- | --- | --- | --- | --- | --- |
| 轨道一 | 单图打分 | 自研工具（能力验收） | 美团业务版（每场景约50条） | 逐条任务，对单张结果图逐题打分 | 底线通过率、信息传达效果分、视觉体验效果分、一次可用率 |
| 轨道二 | 图片排序 | 自研工具 + 竞品（盲测横评） | 通用竞品版（每场景固定20条） | 逐条任务，同时看4张结果图，逐题选最好 | 竞品胜率（总）、各维度胜率 |

---

## 二、题目设计核心原则

### 原则①：全场景固定题目

同一轨道、同一设计场景下，<strong>所有任务使用同一套评测题目</strong>，不针对每条任务单独出题。评测员通过阅读任务提示词获取上下文，题目本身不变。

### 原则②：直接可观察

<strong>题目必须可以通过直接看图回答</strong>，不允许要求评测员想象效果（如"想象缩小到32px后是否清晰"）。需要将想象类标准转化为可直接观察的代理指标：

| 原始标准 | 转化后的可观察题目 |
| --- | --- |
| 缩小至32px后是否可识别 | 哪张 logo 图形最简洁，没有太多细碎装饰？ |
| 黑白版是否可辨 | 哪张 logo 图形线条最清晰，形状本身最好认？ |
| 横竖版适配性 | 哪张 logo 的图形和文字排布最整齐均衡？ |

### 原则③：小学生可读

题目语言必须简单直白，避免专业术语。用日常语言描述可直接感知的结果。

### 原则④：维度全覆盖

题目需从客观标准到主观标准完整覆盖，不遗漏任何评测维度。

## 三、轨道二：图片排序题目规范

### 3.1 题目结构

每个设计场景设置一组固定排序题（建议10～14题），覆盖以下四个维度：

| 维度 | 建议题数 | 说明 |
| --- | --- | --- |
| 任务理解 | 2题 | 任务完成度 + 行业感/风格符合度 |
| 基础质量 | 4题 | 清晰度/无缺陷 + 图形简洁 + 形状辨识 + 排版均衡 |
| 信息传达（含品牌适配） | 4题 | 记忆点 + 主次分明 + 行业感 + 受众适配 |
| 视觉体验（含视觉创新） | 2题 | 差异化 + 创意设计感 |

### 3.2 评测选项

所有题目统一使用以下选项，<strong>不因题目类型不同而增减</strong>：

```plain text
A / B / C / D / 差不多 / 此题目与任务无关
```

| 选项 | 含义 | 计分规则 |
| --- | --- | --- |
| A / B / C / D | 该产品最好 | 该产品得 1 分，其他得 0 分 |
| 差不多 | 无明显差异 | 所有产品各得 0.5 分 |
| 此题目与任务无关 | 该题与当前任务无关（常见于编辑型任务） | 该题跳过，不计入分子和分母 |

> <strong>"此题目与任务无关"使用场景</strong>：编辑型任务（如仅改品牌名）中，涉及创意差异性、行业风格等维度的题目可能与任务核心无关，此时选此项跳过，不影响得分。

### 3.3 胜率计算

```plain text
某产品维度胜率 = 该产品在该维度被选最好的次数 ÷ 实际参与计分的题数 × 100%
```

<strong>分母为动态值</strong>：选"此题目与任务无关"的题目从分母中剔除。

| 指标 | 题目范围 |
| --- | --- |
| 竞品胜率（总） | 全部题目 |
| 任务理解维度胜率 | 任务理解类题目 |
| 基础质量维度胜率 | 基础质量类题目 |
| 信息传达维度胜率 | 信息传达 + 品牌适配类题目（合并计算） |
| 视觉体验维度胜率 | 视觉体验 + 视觉创新类题目（合并计算） |

## 四、轨道一：单图打分题目规范

### 4.1 题目结构

每个设计场景设置一组固定打分题，分为客观题（Pass/Fail）和主观题（0-2分）两部分。

| 类型 | 评分方式 | 对应维度 |
| --- | --- | --- |
| 客观题 | 是 / 否（Fail即触发底线） | 任务理解与执行、基础专业质量、内容合规性 |
| 主观题 | 0 / 1 / 2 分 | 信息传达效果、视觉体验效果 |

### 4.2 底线判定规则

- 任意客观题选"否"，该任务判定为 <strong>Fail（不可用）</strong>

- 主观题仅在客观全部通过后才有意义

### 4.3 指标计算

```plain text
底线通过率 = 该维度全部Pass的任务数 ÷ 总任务数 × 100%
效果分     = 所有Pass任务在该维度的得分均值（0-2分）
一次可用率  = （客观全Pass 且 主观均分≥1.0 的任务数）÷ 总任务数 × 100%
```

## 五、文件命名与目录规范

```plain text
评测题目设定/
└── [场景名]/
    ├── 轨道一_单图打分题目.md    # 固定打分题 + 全部任务列表 + 计算说明
    └── 轨道二_图片排序题目.md    # 固定排序题 + 全部任务列表 + 胜率计算说明
```

- 每个文件分三部分：① 固定评测题目表；② 评测任务列表（含空白答题列）；③ 得分/胜率计算说明

- 已完成场景：Logo设计（轨道一 + 轨道二均已完成）

## 六、各场景题目设计要点

不同设计场景的主观维度差异较大，出题时需参照该场景的评测标准，重点关注：

| 场景 | 主观维度特色 | 需重点设计的题目方向 |
| --- | --- | --- |
| Logo | 记忆点、行业属性、创意差异性 | 一眼记住、行业感、跟同类不一样 |
| IP形象 | 品牌个性、情感传达、受众吸引力 | 有没有角色感、可爱/活泼感 |
| 线上营销 | CTA突出、1秒识别、社交传播力 | 主标题醒目、折扣好不好找 |
| 商品美化 | 食欲感、购买欲望、价格醒目 | 想不想点进去买、食品看起来好吃吗 |
| 线下推广 | 远距离可读性、印刷色彩 | 3米外能读到关键信息吗 |
