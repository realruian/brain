## 一、评测目标及评测模式

### 1、两种评测模式的区别

| 维度 | 内部业务评测 | 竞品 SBS 评测 |
| --- | --- | --- |
| <strong>目的</strong> | 追踪自研 Agent 在业务场景的质量趋势 | 横向对比自研与竞品的相对表现 |
| <strong>评测方式</strong> | 单图评测：专家看「任务要求 + 结果图」答题 | 多图排序：专家盲测，对多方结果按任务目标排序 |
| <strong>问题类型</strong> | 通用固定（信息传达 3 项 + 视觉体验 3 项，加权打分） | 任务专属（每道评测题预配置对应的 SBS 问题） |
| <strong>输出指标</strong> | 0-2 分 → 汇总为一次可用率 | 胜率（自研排第一的比例） |
| <strong>盲测要求</strong> | 不需要 | 必须（评测员不知道哪个是自研、哪个是竞品） |

### 2、指标与受众对应关系

| 受众 | 核心关注指标 |
| --- | --- |
| 管理层 / 汇报 | 一次可用率、竞品胜率 |
| 产品 / 研发团队 | 三项底线通过率、底线稳定率@3、信息传达效果分、视觉体验效果分 |
| 竞品对比报告 | 竞品胜率、各维度均分对比 |

### 3、效果评估指标计算关系

```plain text
【内部业务评测】

底线通过率（总）
  ├── 任务理解底线通过率（人工）
  ├── 基础质量底线通过率（自动 + 人工）
  └── 合规底线通过率（自动 + 人工）

底线稳定率@3（抽取 20% 任务，相同 prompt 跑 3 次）

主观评分（仅底线全 Pass 的任务参与）
  ├── 信息传达效果分（3 项加权打分 → 0-2 分）
  └── 视觉体验效果分（3 项加权打分 → 0-2 分）

↓ 自动汇总

一次可用率 = 底线 Pass 且主观均分 ≥ 1.0 的任务 / 总任务

---

【竞品 SBS 评测】

同一评测任务集 → 各方（自研 + 竞品）独立生成结果
↓
专家盲测，按任务预配置的 SBS 问题排序
↓
竞品胜率 = 自研排第一的任务 / 总任务
```

## 二、效果评估指标操作指引（评测员执行参考）

### 1、底线通过率

<strong>衡量的是什么：</strong> AI 生成的设计结果是否满足最基本的可用条件。

这是一个"合格线"指标，任何触发底线的结果都被判定为不可上线使用，也不参与后续主观质量评分。底线通过率由三个子维度构成，分别衡量不同类型的基础问题。

评测员对照任务要求和结果图逐条核查，触发任意一条 → 该维度记 Fail。

此外，为避免模型偶发触及底线问题，新增底线稳定率指标。通过抽查底线任务，查看产品能否在 3 次测试中均通过评测。

#### 1.1 任务理解底线（人工核查）

是否正确理解了用户需求并按要求执行。

触发这条底线，意味着模型"没听懂"或"没做对"——例如要求生成 3 张图只生成了 2 张、要求端午节风格却生成了春节风格、指定包含品牌 Logo 但结果里没有。

| 核查项 | 判定说明 |
| --- | --- |
| 数量是否满足 | 需求要求的图片数/元素数是否完整？（如需 3 张但生成 2 张 → Fail） |
| 尺寸是否满足 | 输出尺寸与需求规格是否一致？（允许 ±5% 误差） |
| 指定对象是否出现 | 需求中明确要包含的元素/产品/人物是否均已出现？ |
| 是否存在漏项 | 需求中有明确要求但未执行的内容？（如"含价格信息"但未出现） |
| 是否存在跑题 | 生成主题与需求是否明显偏离？（如需求端午节但生成了春节风格） |

#### 1.2 基础质量底线（自动 + 人工）

生成结果是否达到基础的视觉质量标准，没有明显的技术性错误。

触发这条底线，意味着图片本身存在硬伤——例如文字写错了、人物肢体变形、图像模糊不清、核心元素被大面积遮挡。这类问题会直接导致内容无法使用。

<strong>自动检测（脚本先跑，触发即 Fail）：</strong>

| 检测项 | 工具 | Fail 条件 |
| --- | --- | --- |
| 错字 / 文案不符 | PaddleOCR | 识别结果与需求文案存在错字、缺字、多字 |
| 分辨率不达标 | OpenCV | 输出分辨率低于需求规格 |

<strong>人工判定（自动检测全通过后再核查）：</strong>

| 核查项 | 判定说明 |
| --- | --- |
| 主体形变 / 崩坏 | 人物面部、肢体、产品外观是否出现明显结构错误？ |
| 核心元素遮挡 | 主要元素是否被大面积遮挡（超过约 30%）？ |
| 画面清晰度 | 是否存在明显噪点、模糊、压缩失真？ |

#### 1.3 合规底线（自动 + 人工）

生成结果是否符合品牌规范和内容规范，没有违规风险。

触发这条底线，意味着内容存在合规问题——例如品牌 Logo 用错了、出现了竞品元素、包含政治敏感或违禁内容。合规问题在业务场景中是一票否决项。

<strong>自动检测：</strong>

| 检测项 | 工具 | Fail 条件 |
| --- | --- | --- |
| 品牌色值偏差 | 色值检测脚本 | 主色与品牌标准色色差 ΔE > 10 |

<strong>人工判定：</strong>

| 核查项 | 判定说明 |
| --- | --- |
| IP / Logo 正确性 | 品牌 IP 或 Logo 是否正确出现且无变形？ |
| 业务元素正确性 | 二维码可扫描？商品图为指定商品？ |
| 违禁内容 | 是否出现政治敏感、违禁、歧视等内容？ |
| 竞品元素 | 是否出现竞品 Logo、未授权水印？ |

#### 1.4 底线稳定率

<strong>衡量的是什么：</strong> 相同的任务重复执行多次，底线是否能稳定通过。

底线通过率衡量的是"能不能做到"，底线稳定率衡量的是"能不能稳定做到"。即使某个任务偶尔能通过底线，如果重复三次有时通过、有时不通过，说明模型输出不稳定，实际使用时用户体验会受到明显影响。

```plain text
底线稳定率@3 = 抽查任务中 3 次均通过底线的任务数 / 抽查任务总数 × 100%
              （抽取 20% 任务，相同 prompt 各跑 3 次）
```

### 2、信息传达效果分（内部评测，3 项加权打分）

<strong>衡量的是什么：</strong> 设计结果能否清晰、快速地将关键信息传递给用户。

这个维度关注的是"功能性"——设计是为了让用户看懂、记住核心信息，而不只是好看。得分范围 0-2 分，目标值 ≥ 1.5。由三个子维度加权构成：

<strong>第1步：每位评测员同时看「任务要求 + 结果图」，对以下 3 个子维度分别选择一个评分</strong>

| 子维度 | 权重 | 好（2分） | 一般（1分） | 差（0分） |
| --- | --- | --- | --- | --- |
| 核心信息识别速度[^1] | 40% | 遮住任务要求，3 秒内能清楚说出图的核心信息 | 大概能说出，但需 5 秒以上或表述不准确 | 无法判断图要传达什么 |
| 视觉主次层级[^2] | 40% | 最重要信息的字号或视觉权重明显高于次级信息，分层清晰 | 有层级但不明显，主次难以区分 | 各元素视觉权重相近，无主次 |
| 关键信息可读性[^3] | 20% | 主要文字在正常观看距离下，不需用力辨认即可清晰阅读 | 需仔细看才能辨认，部分文字与背景区分不明显 | 主要文字难以阅读或与背景融合 |

<strong>第2步：系统计算该评测员的加权得分（范围 0-2 分）</strong>

`核心信息识别速度得分 × 40% + 视觉主次层级得分 × 40% + 关键信息可读性得分 × 20%`

<strong>第3步：多位评测员各自独立完成第1-2步，取所有人加权得分的均值 = 该任务最终得分</strong>

<strong>注：主观均分阈值说明（信息传达效果分和视觉体验效果分）：</strong>

主观评分范围为 0-2 分，三个锚点分别为：

- <strong>0 分</strong>：所有子维度均为「差」

- <strong>1.0 分</strong>：所有子维度均为「一般」（加权平均恰好落在"一般"档）

- <strong>2.0 分</strong>：所有子维度均为「好」

基于此，各阈值的定义逻辑如下：

| 阈值 | 含义 | 用于 |
| --- | --- | --- |
| ≥ 1.0 | 加权均分不低于"一般"水平，无明显短板，基本可交付 | 一次可用率、2轮达标率 |
| ≥ 1.5 | 加权均分在"一般"与"好"之间偏上，整体表现良好 | 信息传达效果分、视觉体验效果分目标值 |

---

### 3、视觉体验效果分（内部评测，3 项加权打分）

<strong>衡量的是什么：</strong> 设计结果的整体视觉感受是否符合使用场景的需求。

这个维度关注的是"体验性"——设计是否在正确的氛围和风格下呈现，让目标用户产生正确的感受。得分范围 0-2 分，目标值 ≥ 1.5。由三个子维度加权构成：

<strong>第1步：每位评测员同时看「任务要求 + 结果图」，对以下 3 个子维度分别选择一个评分</strong>

| 子维度 | 权重 | 好（2分） | 一般（1分） | 差（0分） |
| --- | --- | --- | --- | --- |
| 场景调性契合度[^4] | 40% | 整体风格与任务场景、目标受众高度匹配（促销应有冲击力、节日应有氛围感、品牌宣传应有专业感） | 基本匹配，有 1-2 处略显违和 | 风格与场景明显不符 |
| 配色协调性[^5] | 30% | 视觉上主导色调克制（可感知的主要颜色不超过 3 种），整体配色和谐舒适 | 颜色略显杂乱或有 1-2 处轻微配色冲突，整体尚可接受 | 颜色过多或存在明显视觉冲突，整体配色混乱 |
| 构图平衡性[^6] | 30% | 主体位于视觉焦点区域，画面各部分视觉重量均衡，无明显空旷或拥挤 | 构图基本合理，有轻微失衡感 | 明显失衡（头重脚轻 / 拥挤 / 大面积无效空白） |

<strong>第2步：系统计算该评测员的加权得分（范围 0-2 分）</strong>

`场景调性契合度得分 × 40% + 配色协调性得分 × 30% + 构图平衡性得分 × 30%`

<strong>第3步：多位评测员各自独立完成第1-2步，取所有人加权得分的均值 = 该任务最终得分</strong>

<strong>注：主观均分阈值说明（信息传达效果分和视觉体验效果分）：</strong>

主观评分范围为 0-2 分，三个锚点分别为：

- <strong>0 分</strong>：所有子维度均为「差」

- <strong>1.0 分</strong>：所有子维度均为「一般」（加权平均恰好落在"一般"档）

- <strong>2.0 分</strong>：所有子维度均为「好」

基于此，各阈值的定义逻辑如下：

| 阈值 | 含义 | 用于 |
| --- | --- | --- |
| ≥ 1.0 | 加权均分不低于"一般"水平，无明显短板，基本可交付 | 一次可用率、2轮达标率 |
| ≥ 1.5 | 加权均分在"一般"与"好"之间偏上，整体表现良好 | 信息传达效果分、视觉体验效果分目标值 |

### 4、一次可用率 & （自动汇总）

<strong>衡量的是什么：</strong> 不经过任何修改，第一次生成就可以直接使用的任务比例。

这是一个综合性指标，同时要求：底线全部通过 + 信息传达效果分和视觉体验效果分的均值 ≥ 1.0（即不低于"一般"水平）。

这是最直接反映"产品好不好用"的核心指标。用户最理想的状态是"一次生成就可以用"，不需要反复修改调整。

```plain text
一次可用率   = （底线全 Pass 且主观均分 ≥ 1.0 的任务数）/ 总任务数 × 100%
```

---

### 5、竞品胜率（竞品 SBS 评测）

<strong>衡量的是什么：</strong> 在同一任务下，与竞品结果并排对比时，自研产品被专家选为更优的比例。

SBS（Side-by-Side）是业界常用的竞品对比方法：把自研和竞品的结果并排展示，评测员在不知道来源的情况下（盲测），依据任务目标进行排序。

胜率反映的是<strong>相对竞争力</strong>，而不是绝对质量——即使自研得分不高，只要比竞品更好，胜率就高。

<strong>评测员操作</strong>：同一任务的多方结果并排展示（盲测，不知来源），依据该任务在评测任务集中预配置的 <strong>SBS 问题</strong>进行排序。

<strong>SBS 问题由各场景评测任务集统一配置</strong>，每道评测题对应一个锚定任务目标的专属问题，例如：

| 场景 | SBS 问题示例 |
| --- | --- |
| 活动促销海报 | 哪张图最能让用户第一眼注意到折扣信息？ |
| 活动促销海报 | 哪张图最能让你有点进去了解活动的冲动？ |
| Logo 设计 | 哪个 Logo 看一眼之后最容易记住？ |
| IP 形象设计 | 哪个 IP 形象最贴近目标用户的审美？ |
| 线下推广物料 | 哪张物料在远处（3 米外）看时信息最清晰？ |

```plain text
竞品胜率 = 自研被排第一的任务数 / 总任务数 × 100%
```

## 注释

[^1]: 核心信息识别速度：户第一眼扫过去，能否在 3 秒内抓住这张图想表达的核心信息。

[^2]: 视觉主次层级：画面中最重要的信息，是否在视觉上最突出——通过字号大小、颜色对比、位置等设计手段体现主次关系。

[^3]: 关键信息可读性：主要文字在正常观看条件下能否被轻松阅读，不需要用力辨认。

[^4]: 场景调性契合度：整体风格是否符合任务场景和目标受众的期待。例如促销海报应有紧迫感和冲击力，节日物料应有氛围感，品牌宣传应有专业感。

[^5]: 配色协调性：颜色搭配是否克制、和谐，没有明显的视觉冲突。视觉上主导的颜色是否控制在合理范围内（不超过 3 种），整体看起来舒适而非杂乱。

[^6]: 构图平衡性：画面各部分的视觉重量是否均衡，主体是否位于合理的视觉焦点区域，没有明显的空旷感或拥挤感。
