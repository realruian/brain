## 产品设计验收 · 可上线标准

> 覆盖新生产范式（无设计稿）与旧生产链路（有设计稿）的完整验收准入体系，包含视觉还原、交互体验、适配兼容、内容规范四大维度。

---

### 一、核心原则

验收标准的本质是「用户感知质量的最低保障线」，而非追求完美。**可上线 = 用户不会因为设计问题产生困惑、受阻或负面感知。**

所有维度均设置明确的「阻断项（Blocker）」与「建议项（Advisory）」：

- 🚫 **阻断项**：必须清零方可上线
    
- ⚠️ **建议项**：可在下一迭代修复
    
- 🤖 自动化检测优先，👁️ 人工兜底覆盖盲区
    

---

### 二、两条验收路径

#### 🤖 新生产范式（无设计稿）

AI Coding 生成代码，无设计稿作为视觉基准。验收以「规范知识库」为语义底座，通过自动化工作流（Git Diff 扫描 → 结构树提取 → 规范索引 → 评分）完成主要检测，人工负责交互体验和状态覆盖的兜底。

**可上线准入标准（全部满足）：**

1. **规范合规率 ≥ 90%**：原子层 + 分子层 + 场景层三层均需达标，任一层不达标即阻断。来源：知识库自动检测。
    
2. **设计标准问题 = 0 个**：含操作反馈缺失、热区不达标、标题符号违规、提示文案不规范等。来源：知识库检测。
    
3. **已完成大屏适配，且无布局适配 Bug**：大屏（428pt+）无错位，小屏（320pt）无截断。来源：图片比对。
    
4. **逻辑/流程/性能验收 Badcase = 0 个**：含空态/错误态/加载态。来源：系统自动识别 + 人工反馈。
    
5. **真实数据填充后无文案截断/溢出；关键异常态无设计标准问题**：含空态、加载态、错误态、权限受限态的文案与布局。
    

#### 🎨 旧生产链路（有设计稿）

人工设计稿驱动研发，有明确的视觉基准。验收以「设计稿还原度」为核心，通过图片比对 + 结构树节点匹配量化偏差。新容器（洋葱）可达到数据层完全一致，对比准确度无风险；旧容器依赖当前探索方案进展，置信度需在报告中标注。

**可上线准入标准（全部满足）：**

1. **标准机型（375pt）设计稿还原度 = 100%**：来源：图片比对。新容器数据层完全一致；旧容器依探索方案进展，置信度需标注。
    
2. **设计标准问题 = 0 个**：含操作反馈缺失、热区不达标、标题符号违规、提示文案不规范等。来源：知识库检测。
    
3. **已完成大屏适配，且无布局适配 Bug**：大屏（428pt+）无错位，小屏（320pt）无截断，异形屏安全区适配。来源：图片比对。
    
4. **逻辑/流程/性能验收 Badcase = 0 个**：含完整用户路径验证。来源：系统自动识别 + 人工反馈。
    
5. **真实数据填充后无文案截断/溢出；关键异常态符合设计稿**：设计稿中的空态/错误态/加载态需一并验收，不可仅验收正常态。

**还原度偏差分级处理（旧链路）：**

|偏差类型|示例|处理方式|是否阻断|
|---|---|---|---|

### 四、阻断项汇总（Blocker · 必须清零方可上线）

**视觉 & 规范类阻断：**

- 有稿场景：标准机型下存在可感知的视觉偏差（字色错误、圆角缺失、间距明显偏移等）
    
- 无稿场景：规范合规率 < 90%（原子/分子/场景层任一维度）
    
- 使用了非规范色值、非规范字号（未走 Token 体系）
    
- 组件使用错误（如用了错误变体或已废弃组件）
    

**交互 & 体验类阻断：**

- 热区 < 44pt 导致用户无法正常点击
    
- 操作无反馈（点击无响应、加载无状态）
    
- 页面跳转/返回逻辑错误，用户被困在错误状态
    
- 大屏/小屏下出现内容遮挡、截断、布局错乱
    
- 真实数据填充后文案溢出容器或被截断
    

---

### 五、验收流程

1. **触发验收**：研发提测 / AI Coding 完成
    
2. **自动化检测**：知识库扫描 + 图片比对
    
3. **生成报告**：问题分级 + 定位代码
    
4. **人工复核**：交互/体验/状态覆盖
    
5. **修复迭代**：阻断项清零后重跑
    
6. **准入上线**：全部阻断项 = 0
    

---

### 六、上线结论判定

**✅ 准入上线（满足以下全部条件）：**

- 所有 P0 阻断项 = 0
    
- P1 项已修复或有明确修复计划（本迭代内）
    
- 验收报告已生成并标注置信度
    
- 人工复核交互/状态覆盖已完成
    

**🚫 阻断上线（存在以下任一情况）：**

- 任一 P0 阻断项未清零
    
- 新范式：任一层规范合规率 < 90%
    
- 旧链路：标准机型存在可感知视觉偏差
    
- 大屏/小屏存在布局 Bug 或内容截断
    
- 关键交互路径存在 Badcase
    

> 💡 **关于 P2 建议项**：P2 项不阻断上线，但需在验收报告中记录，并在下一迭代的需求排期中纳入修复计划。连续 3 个迭代未修复的 P2 项，自动升级为 P1 项。
