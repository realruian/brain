# Obsidian 日常使用

## 笔记怎么写

不用追求完美，几个关键词比空白页有用一万倍。格式参考：

```markdown
---
tags:
  - 某个标签
date: 2026-04-14
---

# 标题

正文内容
```

判断标准：**这件事将来可能还会遇到吗？** 是就记，不是就别记。

## 笔记放哪里

| 内容类型 | 放哪里 | 举例 |
|----------|--------|------|
| 随手一记，还没想好分类 | `00-Inbox` | "今天看到一个不错的 Canvas API 用法" |
| 跟某个项目直接相关 | `01-Projects` | "design-studio 架构决策" |
| 长期关注的领域知识 | `02-Areas` | "前端性能优化"、"AI 工具链" |
| 参考资料、教程 | `03-Resources` | "Next.js Server Actions 用法" |
| 完成或不再需要 | `04-Archive` | 已结束项目的笔记 |
| 每日记录 | `05-Daily` | "2026-04-14" |

**拿不准放哪？先扔 `00-Inbox`。** 定期让 Claudian（Obsidian 中集成 Claude Code 的插件，在侧边栏提供对话界面）帮你归档：

```
看一下 00-Inbox 里的笔记，帮我分类移到合适的文件夹
```

## 笔记怎么连接

用双向链接把相关笔记串起来：

- 提到另一篇笔记时用 `[[笔记名]]`
- 给笔记打 tags 方便搜索和筛选
- 不用刻意建链接，写的时候自然联想到就加

## 常用 Claudian 笔记指令

| 场景 | 指令 |
|------|------|
| 整理 Inbox | 看一下 00-Inbox 里的笔记，帮我分类移到合适的文件夹 |
| 搜索笔记 | 我之前记过关于 XX 的内容，帮我找一下 |
| 生成周总结 | 看一下 05-Daily 里这周的笔记，帮我生成周总结 |
| 创建参考笔记 | 帮我在 03-Resources 里创建一篇关于 XX 的参考笔记 |

## 核心原则

1. **别让记笔记变成负担** — 能交给 Claudian 的就交给它
2. **先扔 Inbox，再整理** — 别纠结分类
3. **只记将来有用的** — 不是所有事都值得记录
4. **笔记不求完美** — 先写下来，以后再补充
