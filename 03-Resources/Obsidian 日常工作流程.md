## 每日三个时刻

### 1. 开工前（2 分钟）

打开 Obsidian，在 Claudian 侧边栏输入：

```
看一下 ~/dev/meituan/design-studio 的 git status 和最近的 TODO，告诉我今天该干什么
```

作用：快速回忆昨天的进度，明确今天的方向。

### 2. 干活中（随手记）

遇到以下情况时，切到 Obsidian 写几个关键词：

- **踩坑**："PSD 解析器在图层超过 50 层时 OOM，改成流式处理解决了"
- **技术决策**："选了方案 B，因为方案 A 在并发场景下有竞态问题"
- **学到新东西**："Next.js 16 的 Server Actions 可以直接返回流式响应"
- **待办**："截图导出模块需要兼容 Retina 屏幕"

判断标准：**这件事将来可能还会遇到吗？** 是就记，不是就别记。

记录方式：

- 简单的直接写在当天的 Daily Note 里（`05-Daily`）
- 重要的技术决策单独建一个笔记放到 `01-Projects` 对应项目下

### 3. 收工前（1 分钟）

在 Claudian 侧边栏输入：

```
看一下 ~/dev/meituan/design-studio 今天的 git log，帮我写个今日总结到 05-Daily 里，文件名用今天的日期
```

Claude Code 会自动读 commit 记录生成总结。你扫一眼，有遗漏的补一句就行。

## 常用 Claudian 指令

| 场景     | 指令                                          |
| ------ | ------------------------------------------- |
| 生成项目文档 | 读一下 ~/path/to/project 的代码结构，帮我创建项目笔记        |
| 更新项目笔记 | 看一下 design-studio 最近一周的 git log，更新项目笔记的进展记录 |
| 记录技术决策 | 帮我在 01-Projects 里记录一个技术决策：为什么选 XX 方案        |
| 整理零散笔记 | 看一下 00-Inbox 里的笔记，帮我分类移到合适的文件夹              |
| 搜索历史记录 | 我之前记过关于 PSD 解析的坑，帮我找一下                      |
| 周报总结   | 看一下 05-Daily 里这周的笔记，帮我生成周报                  |

## 笔记去哪里

|内容类型|放哪里|举例|
|---|---|---|
|随手一记，还没想好分类|`00-Inbox`|"今天看到一个不错的 Canvas API 用法"|
|跟某个项目直接相关|`01-Projects`|"design-studio PSD 解析优化方案"|
|长期关注的领域知识|`02-Areas`|"前端性能优化"、"AI 工具链"|
|参考资料、教程|`03-Resources`|"Next.js Server Actions 用法"、本文档|
|完成或不再需要|`04-Archive`|已结束项目的笔记|
|每日总结|`05-Daily`|"2026-04-14"|

## 核心原则

1. **别让记笔记变成负担** — 能交给 Claude Code 的就交给它
2. **先扔 Inbox，再整理** — 别纠结分类，定期让 Claude Code 帮你归档
3. **只记将来有用的** — 不是所有事都值得记录
4. **笔记不求完美** — 几个关键词比空白页有用一万倍
