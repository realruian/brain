## 基本操作

**启动：** 先 `cd` 到你的项目目录，然后输入 `claude`

```bash
cd ~/my-project
claude
```

Claude Code 会自动读取你整个项目的结构。

- **发消息：** 直接打字，按 Enter 发送。就像聊天一样，但它能看到你的文件。
- **多行输入：** 按 `\ + Enter` 换行，这样不会直接发送，可以写多行再一起提交。
- **退出：** 输入 `exit` 或按 `Ctrl+D`
- **中断 Claude：** 按 `Esc`，Claude 会停下正在做的事

## 最常用的快捷键

| 按键              | 作用                               |
| --------------- | -------------------------------- |
| `Enter`         | 发送消息                             |
| `Esc`           | 中断 Claude                        |
| `Esc Esc`（连按两次） | 撤销 Claude 最近的改动                  |
| `↑`             | 翻出上一条你发的消息                       |
| `Tab`           | 自动补全                             |
| `\ + Enter`     | 多行输入（换行但不发送）                     |
| `Shift+Tab`     | 切换权限模式                           |
| `Option+P`      | 快速切换模型（macOS）                    |
| `Ctrl+O`        | 切换 Focus View（只显示 prompt + 最终回复） |
| `Ctrl+L`        | 清屏                               |
| `?`             | 查看所有可用快捷键                        |
| `/`             | 查看所有斜杠命令                         |
| `!` + 命令        | 运行终端命令并把结果给 Claude 看             |

## 斜杠命令

在 Claude Code 里输入 `/` 就能看到所有命令。最常用的：

| 命令                 | 干嘛的                               |
| ------------------ | --------------------------------- |
| `/help`            | 查看帮助                              |
| `/model`           | 切换模型（Opus 4.6 / Sonnet 4.6）       |
| `/effort`          | 切换思考深度（low ○ / medium ◐ / high ●） |
| `/status`          | 查看当前状态（模型、账号、MCP 等）               |
| `/cost`            | 查看本次会话花了多少                        |
| `/clear`           | 清空上下文，从头开始（换任务时用）                 |
| `/compact`         | 压缩上下文（上下文快满时也会自动触发）               |
| `/rewind`          | 撤回到之前某个时间点                        |
| `/init`            | 初始化项目，自动生成 CLAUDE.md              |
| `/review`          | 对当前改动做代码审查                        |
| `/copy`            | 交互式选择代码块复制到剪贴板                    |
| `/loop`            | 循环执行任务，如 `/loop 5m /review`       |
| `/hooks`           | 查看和配置 Hooks                       |
| `/mcp`             | 查看 MCP 服务连接状态                     |
| `/login` `/logout` | 登录 / 登出                           |

## 日常使用场景

**1. 让 Claude 了解你的项目（每次新会话第一件事）**

```
这个项目是什么？帮我梳理一下整体架构
```

**2. 写功能**

```
在 src/components 里新建一个 Button 组件，支持 primary / secondary 两种样式，用 TypeScript
```

**3. 修 Bug**

```
运行 npm run build 报了一个 TypeScript 错误，帮我修
```

**4. 引用特定文件（用 @）**

```
看一下 @src/api/client.ts 这个文件，帮我理解请求封装的逻辑
```

**5. 跑命令并把结果给 Claude**

直接输入 `!npm test`，Claude 会看到命令输出并据此帮你解决问题。

**6. Plan Mode（先规划后执行）**

```
帮我规划一下怎么给这个项目加国际化支持，先不要改代码，只给我方案
```

看完方案满意了再说"按这个方案执行"。

## 权限系统

Claude 在执行文件修改或命令时，会根据你底部状态栏的权限设置来决定要不要问你。用 `Shift+Tab` 在前三种模式间循环切换：

| 模式 | 说明 |
|------|------|
| **Default** | 每步都要确认，最安全，新手推荐 |
| **Auto-accept edits** | 改文件自动通过，跑命令仍需确认 |
| **Plan mode** | 只读不动，先分析再决策 |
| **Auto mode** | 用独立的安全分类器自动批准低风险操作，阻止高风险操作（如 force-push、生产部署）。需 Team 计划 |

## CLAUDE.md（你的项目"说明书"）

在项目根目录创建一个 `CLAUDE.md` 文件，Claude Code 每次启动都会自动读取。最快的方式是用 `/init` 让 Claude 帮你生成第一版，然后手动精简。

每次 Claude 犯了一个错，就加一行进去，防止下次再犯。

> CLAUDE.md 控制在 200 行以内，只放普遍适用的规则（编码风格、常用命令、不能动的文件）。具体项目的编写规范可以参考官方文档。

---

## Skills（扩展 Claude 的能力）

在 `.claude/skills/` 下创建 `SKILL.md` 文件，文件名即 `/` 命令名。Claude 在相关任务中会自动加载，不会每次都占上下文。也可以用 `/skill-name` 手动触发。

> `.claude/commands/` 仍然兼容，但 Skills 是 2026 年推荐的方式，支持更多功能（模型覆盖、工具白名单、子代理隔离等）。用 `/plugin` 可以浏览社区插件市场。

## 实用建议

- **换任务时用 `/clear`**：上下文里留着上一个任务的信息会干扰新任务，清掉重新开始
- **用 `/effort high` 处理复杂问题**：简单任务保持 medium，架构设计或疑难 bug 切 high
- **日常用 Sonnet，难题切 Opus**：`/model sonnet` 快而便宜，`/model opus` 慢但深（也可以按 `Option+P` 快速切换）
- **经常 `/cost` 看一眼**：了解你的用量，避免不知不觉用超了
- **善用 `Esc Esc` 撤销**：Claude 改错了，连按两次 Esc 回到改之前的状态
- **用 `Ctrl+O` 开 Focus View**：屏幕只保留 prompt 和最终回复，减少干扰，专注编码
- **`--dangerously-skip-permissions` 慎用**：跳过所有权限确认，仅适合 CI/自动化场景，日常开发别用
