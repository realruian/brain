# 二、“AI友好设计稿”包含什么及原理

**1.主要要做哪些和重要性排序**

基于对**设计师当前作图流程的观察，现阶段存在的问题 + 前端开发的调整难度，AI友好设计稿规范化的重要性排序如下：**

![[../_attachments/Pasted image 20260531173548.png]]

为什么？从原理上来讲：

设计稿中**节点（图层）的分组**（以容器构成），对应代码中的 **分组** （html-容器div），在多前端框架下都是实现视觉的“地基”；

设计稿的 **节点的样式** 影响代码中各容器的 **样式赋值**（css），对应设计习惯，分别可以分成1）决定了空间关系的 **定位** 类，和2）决定细节**视觉表现**（如颜色、字体）的 **样式** 类。

> 💡注解：
> 
> 设计节点(node)：**可以简单理解为图层**，容器、文字、图片、图形等等.....推荐阅读[https://developers.mastergo.com/apis/nodeTypes.html](https://developers.mastergo.com/apis/nodeTypes.html)
> 
> 节点的属性(properties)：**可以简单理解为样式**。宽度、高度、颜色、有没有隐藏、等等......

所以，从「视觉还原度」与「代码质量」两个维度：
**AI友好的设计稿 = 工程化的设计节点分组方式+ 合理的节点样式赋值方式 = 输出好的dsl结构 = 输出好的代码质量**

![[../_attachments/Pasted image 20260531173650.png]]
**2.影响程度较小的规范**

在使用多种mcp导出的测试中，**设计稿结构（嵌套层级）和自动布局是影响D2C生成质量的关键因素**，而有些作图规范如删除隐藏图层（会在导出时被mcp砍掉）、和语义化命名（会被llm修正）等影响较小。

结合代码知识和实践认知，我们总结下文设计稿规范对生成代码质量的影响报告和实践建议。

## 3.2 定位的结构化

① 在模块设计阶段使用盒模型进行结构拆分，同时通过自动布局提高数据的结构化程度

盒模型：https://blog.csdn.net/weixin_39649965/article/details/111665324

通过全自动布局约束盒模型，具有固定间距的区域必须启用 gap；容器内部的留白使用 padding。

![[../_attachments/Pasted image 20260531174751.png]]

**WHY? 结构化边距数值，更方便模型理解你搭建的盒模型；**

如果没有这么绑，模型也会自己计算padding并赋值，但是有很大的算错概率。

bad case：缺少边距约束的盒模型
如左图样式导出，由于缺少中间的容器和卡片的布局约束，会导致导出效果下图：

![[../_attachments/Pasted image 20260531174824.png]]

②  用自动布局提升模块间排版布局的结构化：

在整理时先判断：该容器是否应该是 自动布局？若视觉呈现是规律排列（横排/竖排）或规律布局（局中/居左等），应使用自动布局；

若确实需要自由摆放，再考虑绝对定位。

![[../_attachments/Pasted image 20260531174845.png]]

WHY? 自动布局的会传输css layout相关属性，

使代码自带各元素之间的空间关系；

而没有自动布局时，会退回到 用绝对定位absolute（relativeX / Y）实现无法获取元素之间的排版关系，只有各元素和容器的数值关系，容易导致排版混乱+无响应式布局。

bad case：没有自动布局时元素错误的对齐方式

# 四、常见问题

**Q: ai（llm）起到什么用？**

A: 1）补全代码结构 + 2）结合图片理解布局，添加结构化布局信息。

**Q: 为什么设计稿规范不规范，还原度好像差不多**

A: 两个可能：1）运用了绝对定位，尽管还原了视觉但是代码结构糟糕，完全不可用；2）在还原过程中，被llm兜底，纠正了还原效果；

**Q: D2C是否能100%还原？**

A：不一定能，因为mg（包括figma）和代码实现存在gap，比如mg没有代码中常用的margin的设定、宽高设置也不能一一对应、旋转角度没直接导出等等非常多的差异！但是细节样式相对来说修改比较容易，**规范化设计稿需要做的是避免结构出问题，**然后通过mcp能力建设和优化弥补其他导出问题。

**Q: 响应式(responsive)是怎么实现的？**

A: 用flex布局，内部设置参考如下：

原子级：**flex-grow:1**（设计稿：对原子item选择「充满」）；

容器级别：**align-self: stretch**（设计稿：让item充满容器，再给容器选择「充满」）

**但是，**由于设计稿内宽高属性与代码映射并不一一成立，起辅助作用；大致可以参考

- 适应 = `width: auto` / 某些场景 `width: 100%`（容器限制下）
    
- 充满 =`flex-grow: 1` / `align-self: stretch` / 某些情况下 `display: inline-block`
    
- 固定 = width: 300px
    

![[../_attachments/Pasted image 20260531175606.png]]

**Q: mg的「约束」功能的意义？**

A：“约束 constraint”在 MG 内有操作含义，但对导出代码无直接影响。
