## 一、产品定位与边界

### 1.1 产品定位

**作为美境 AI 设计师的新能力模式**，而非独立产品。外卖设计中心提供外卖垂直的场景、模板资产与知识体系；美境提供平台底座、AI 能力基建与入口集成。两方共建，面向外卖营销运营提供一站式自助化设计能力。

### 1.2 与美境现有能力的关系

- **复用**：美境已有的账号权限、资产管理、通用 AI 能力（生图、图层编辑等）尽可能复用，避免重复建设
- **扩展**：外卖域特有能力（PSD 复杂修改、多尺寸资源位延展、外卖品牌规范校验、行业知识体系）作为美境的新能力模块接入
- **协同**：外卖产出的模板/素材/知识作为资产沉淀在美境，为跨业务复用提供基础

> **美境已有能力 vs 本需求新增能力** `// TODO(美境侧补充 gap 清单)`

### 1.3 功能差异化定位

相对于美境现有的通用能力，本次接入的外卖模块侧重三方面：

- **外卖属性资产建设**：外卖模板库、风格化模型、外卖垂直素材库
- **操作便捷度优化**：降低运营使用门槛，针对外卖工作流做交互定制
- **链路化体验**：一站式生成整个活动素材的端到端链路

### 1.4 与 v1 的差异说明

v1 将本工具定位为短期独立试跑、长期集成美境。v2 反转叙事：**直接作为美境能力共建项目**，不再以独立产品形态存在。v1 中 P6 的"平台化底座"大部分能力由美境承担，本文档不再重复建设，只保留外卖垂直定制部分。

## 二、接入形态方案

接入形态为本次需求评估的**核心待定项**，外卖侧给出三种候选预案，请美境侧共同评估拍板。

### 2.1 方案 A：独立 Tab / 独立模式入口

**描述**：在美境主界面下新增"外卖营销"Tab 或"外卖自助设计"模式，运营从该入口进入专属工作流。

**优势**：
- 外卖工作流独立，可做深度垂直优化
- 数据可独立回收，提效数据清晰归属
- 试点期间可快速迭代，不影响美境主流程

**劣势**：
- 美境入口复杂度增加
- 外卖运营需感知"切换到外卖模式"

### 2.2 方案 B：嵌入现有创作流

**描述**：外卖能力作为美境创作流中的节点或组件，运营在现有创作流中选择"外卖模板"/"外卖资源位"时自动调用对应能力。

**优势**：
- 用户感知一致，学习成本低
- 美境入口不分叉

**劣势**：
- 外卖定制化交互受限于美境现有 UI 框架
- 数据埋点需与美境主链路耦合

### 2.3 方案 C：底层能力被调用

**描述**：外卖能力以 API / Skill 形式暴露，美境主链路按需调用；外卖侧无独立入口。

**优势**：
- 耦合度最低，改造量最小
- 能力可被其他平台复用

**劣势**：
- 缺少承载运营用户习惯的载体
- 外卖场景的链路化体验难以呈现

### 2.4 推荐与待定

外卖侧**倾向方案 A**（独立 Tab）用于一期试点、方案 B+C 用于长期融合。最终形态请美境侧基于平台规划决策。

## 三、功能需求（按期次）

工具按"原子能力 → 场景链路 → 知识沉淀 → 组件迭代 → 平台整合"的节奏推进，共规划 **六期**。每个功能均标注**美境接入点**，标明是复用美境已有能力、扩展美境能力，还是外卖独立建设。

### 3.1 P1：典型修改类原子能力（4W3 ~ 4W4）

**目标**：跑通 1~2 个典型修改类需求素材的全自助化设计流程，业务可自主完成全部素材修改并上线。

#### F1.1 以图拓图（常规资源位拓展）
- **功能描述**：上传一张基础素材图，AI 自动适配到首页运营卡片、固定位 banner、下拉二楼 banner、美团猜喜等常规资源位的不同尺寸/比例
- **输入**：单张原图 + 目标资源位类型（可多选）
- **输出**：多尺寸、构图合理的素材包；支持单张下载与打包下载
- **验收标准**：覆盖 ≥ 4 个常规资源位 × 3 种尺寸；首次生成耗时 ≤ 20s；生图采纳率 ≥ 50%
- **美境接入点**：`// TODO(美境是否已有以图拓图能力，若有则直接复用)`

#### F1.2 标识排序能力
- **功能描述**：对素材中的价格/标签/图标等关键元素进行智能排序、对齐与层级归整，保证视觉信息的主次关系符合营销规范
- **输入**：带元素标签的素材 或 元素描述清单
- **输出**：排序规整后的素材图
- **验收标准**：规则命中率 ≥ 90%；业务侧一次通过率 ≥ 70%
- **美境接入点**：`// TODO(外卖侧新增能力，美境侧提供能力注册入口)`

#### F1.3 复杂资源位 / 动态素材修改能力
- **功能描述**：支持对含多图层、动效帧或复杂构图的资源位素材进行改字、改色、换图等精细化修改（对应 design-studio 现有 PSD 图层编辑器）
- **输入**：源素材文件（PSD）+ 修改指令（文案/颜色/替换图层）
- **输出**：保持图层结构完整的修改后素材
- **验收标准**：修改后素材结构完整性 100%；任务报错率 ≤ 5%
- **美境接入点**：`// TODO(外卖侧提供 PSD 图层编辑器实现，美境侧提供 PSD 存储与运行环境)`

#### F1.4 基础反馈埋点
- **功能描述**：每次生成任务默认带"踩/赞"按钮与反馈话术识别，采集负评率与采纳率
- **验收标准**：全部生成任务 100% 具备反馈入口；负评率可按天回收
- **美境接入点**：`// TODO(美境如有统一埋点体系则接入复用)`

### 3.2 P2：修改类需求全流程打通（4W4 ~ 5W1）

- **依赖**：F1.1、F1.2、F1.3
- **验收标准**：一次生成覆盖 ≥ 5 类资源位；业务端可跳过设计环节直连投放

- **输入**：模板源文件（PSD/Figma/HTML）+ 可编辑字段描述
- **输出**：工具内可被业务选用的模板条目

### 3.3 P3：简单全新 KV 设计流程（5W1 ~ 5W2）

**目标**：产出 3~4 个成功上线案例，半全新设计流程可跑通。

- **输入**：模板选择 + 主视觉素材（可选 AI 生成）+ 活动信息
- **输出**：整套活动 KV 及下延素材

## 五、非功能需求与度量指标

### 5.1 生成质量与效能

- **生图采纳率**：下载素材量 / 已生成的素材量
- **负评率**：被标记为"踩/不满意"的生成任务数 / 有反馈的总任务数
- **累计采纳量**：周期内累计被下载次数
- **默认参数/预设模板直接使用率**：未做任何修改的任务数 / 总任务数
- **优质资产占比**：被复用的资产 / 总资产量

### 5.2 易用性与交互体验

- **首次价值触达耗时**：新用户首次打开工具到第一次成功生成的平均耗时
- **首次任务完成率**：新用户首次登录后无指导完成一次有效生成的比例

### 5.3 用户活跃与粘性

- **留存率**：首次使用用户在第 N 周/月仍在使用的比例
- **人均生成频次**：总生成任务数 / 活跃用户数

### 5.4 系统性能与稳定性

- **任务报错率**：失败任务数 / 发起任务数
- **平均生成耗时**：点击"生成"到第一张预览图完全展示的平均时间

### 5.5 成本与 ROI

- **单次生成算力成本**：总算力成本 / 总生成任务数

## 七、长期规划

### 7.1 AI 认知迭代（越用效率越高）

行业专属语言、业务逻辑、场景规则的自学习与自更新，通过知识图谱的自主构建、补全与修正，将历史对话中的业务术语、行业黑话、场景关联规则内化为模型认知，逐步提升业务语言理解准确性。

### 7.2 让 AI 成为业务增长参谋

打通数据平台实现全链路数据洞察，构建"设计—投放—学习—优化"闭环的数据飞轮。每一次设计投放后的业务效果数据自动回流到模型训练中。

> **共建视角**：数据回流通道由美境主导，外卖侧贡献外卖行业特有的效果指标与归因模型。

### 7.3 外卖模式复制到其他业务域

外卖共建模式跑通后，可作为美境接入垂直业务域的参考范式，向闪购、酒旅、医药、到店等其他设计中心复制。
