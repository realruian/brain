# 面向产品经理的 AI 生成 Demo

# 一、这门课帮你解决什么

**生成 Demo 的常见难点：**

- **不会描述。** 需求在脑子里，但不知道怎么转化为 AI 能理解的指令。写得太笼统 AI 自由发挥，写得太细又不知道从何说起。
- **不合规、不精致。** AI 产出的页面与业务设计语言不一致，或结构对了但细节（间距、层级、状态）达不到交付水平，以及局部修改困难，整体停留在"原型"的阶段。
- **难验收。**缺少明确的质量标准，无法判断产出是否满足一致性和可交付要求。

**课程目标：** 让 非设计同学 能独立完成从需求到 **高保真 Demo** 的全流程——不需要具备设计技能，但能让 AI 产出的页面达到设计师和研发可接手的标准。

**适用人群：** 产品经理为主。设计师建议同步了解协作部分，理解 PM 产出物的质量标准和交接方式。

# 三、怎么把你想要的说清楚

## 3.1 一个万能的 Prompt 模板

把 Prompt 写好，AI 产出的东西就贴近你的预期；Prompt 写不好，怎么调都不对。一个可复用的 Prompt 模板：

【页面类型】+【核心结构】+【关键元素】+【关键交互】+【约束条件】
拆开来说：
- 页面类型：告诉 AI 这是什么页面（列表页、详情页、表单页、看板页……） 
- 核心结构：页面从上到下分几个区域，每个区域是什么 关键元素：每个区域里有什么具体内容（字段、按钮、图表）
-  关键交互：点击什么会发生什么（筛选、跳转、弹窗、展开收起） 
- 约束条件：组件库、尺寸、间距、配色等硬性要求

**为什么要这样分？**

因为 AI 在单轮对话中能处理的有效信息量有限。先让它把大结构做对，再逐步补细节——每一轮你都能确认上一步是对的再继续，避免跑偏后推倒重来。

**PRD 里的描述能不能直接当 prompt 用？**

大部分不能。PRD 描述的是"业务规则"和"功能逻辑"，prompt 需要的是"页面结构"和"视觉呈现"。两者的视角不同，需要做一次转化。具体的转化方法在后面的案例里展开。

## 3.3 写多细才够？四档精准度对比

同一个页面，不同精准度的 prompt 出来的效果差异巨大；不需要每次都写到"非常精准"。Demo 级用"中等"就够了；交付级建议写到"精准"，尤其是涉及组件选择和布局规则的部分。

**知识点 1：组件库选择比 prompt 更重要**

- 这一步是 L1 和 L2 最本质的区别。用 Roo / MTD / Leeze 组件库生成的页面天然贴近内部规范；不选组件库，AI 默认用 shadcn，好看但设计师和研发都不认。
- 怎么判断"是否符合规范"：可以自查下，布局是否跟同类线上页面一致？组件是否用对（该用 Table 还是 List）？间距和字号是否在规范范围内？配色是否跟系统主题一致？

知识点 2：L1的程度建议生成 html 代码

- AI 生成 html 比生成图像更擅长、更可控。图像模型有两个硬伤（文字糊、细节漂），而且不可编辑；

**知识点：协作流程--PM 生成页面，设计师 Review**

- **传统流程：** PM 写 PRD → 设计师从零出稿 → 评审 → 修改 → 交付研发
- **新流程：** PM 用 AI 生成 L3 页面 → 设计师 Review → 修改 → 交付研发

需要注意的是：**"规范合规"和"设计合理"是两件事。** 组件用对了、间距符合规范了，不等于这个页面的设计是好的。还需要关注：信息架构是否合理、交互模式是否恰当、用户体验是否流畅。

# 七、常见问题

| **工具很多，怎么判断符合工作要求？** 工具在快速迭代，今天的选择半年后可能就变了。但判断一个工具"是否可用"的标准不变：它生成的页面能不能符合设计规范？如果一个工具支持加载内部组件库、支持喂入视觉参考、支持精确到像素的样式调整——那它就能做到交付级，非常可用。 |
| --- |
| **一个 prompt 写多长合适？** 没有固定长度，但有一个原则：一轮 prompt 只解决一个层次的问题。如果你发现一轮 prompt 里既在描述结构、又在指定颜色字号、还在补边界状态——那就拆成三轮。AI 在单轮中能处理的有效信息量有限，混着写容易丢信息。 |
| **AI 生成的页面能不能直接交给研发？** 要看做到了哪一层。如果只做到 L1（低保真原型），不能直接交——研发拿到只能当参考看。做到 L2（规范页面），设计师可以接手深化，研发可以参考结构或摘取代码。做到 L3（生产级代码），在真实仓库中且符合代码规范的情况下，研发可以直接在上面开发。但不管哪一层，AI 生稿交付的都是"页面结构和样式"，业务逻辑、接口对接、状态管理仍然是研发的工作。 |
| **生成的代码能直接给研发用吗？** L2 不能直接 merge，但可以作为"设计参考 + 代码参考"交付。L3（通过 NoCode 出的）可以直接提 PR，但研发仍需 review。 |
