# 【评测任务集规范】

<strong>目标：</strong>围绕 AI 设计师中的主要能力场景构建评测集，通过分析通过评测集跑出的结果，明确当前能力是否满足上线要求。

<strong>问题：</strong>

- 真实性差：既往构建的评测集多数以构造的数据为主，并没有结合真实需求场景出发进行构建，真实性存在问题；

- 时效性差：此外，历史的评测集多用于模型微调使用，以服务当时的工具能力为主，当工具能力提升后，评测集不再符合当前的评测情景；

<strong>挑战：</strong>

- 需要从真实业务场景出发，思考 Agent 的底层能力，以此构建一个真实性高，时效性强的评测任务集；

- 还要考虑不同设计场景中的能力要求，在评测集中进行差异化处理。

## 注释

[^1]: 模糊需求：用户的任务请求相对模糊，需要考验模型工具的理解能力

[^2]: 明确任务：用户的任务请求非常具体，对于画面各部分都有明确的要求

[^3]: 编辑型需求：图文指令编辑类型的需求

[^4]: 复合型需求：在一个需求中隐含多个任务请求，比如“生成一个ip形象，并同时根据ip形象生成多个风格不同的表情包/冰箱贴”，这个需求中需要先生成一个ip形象，然后再根据ip形象生成多个不同的延展图像。
