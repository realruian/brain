### 1、背景及目标

#### 1.1 问题及挑战

当前智能创作 Agent在评估上面临以下典型挑战

- **缺乏成熟的评估标准**：Agent评估是业界面临的难点，创作Agent由于输入输出都包含图像导致评估更难，难以找到一个准确的评估标准和Benchmark，
    

- **通用图像评估&业务标准存在GAP**：虽然可以一定程度复用文生图的图像评估方法，评价最终创作结果的美学、指令跟随、文字、合理性等维度，但这些维度和业务要求存在GAP，无法满足业务自定义规则
    
    > [AI设计师-袋鼠IP 问题反馈](https://km.sankuai.com/collabpage/2737063621)中反馈的问题都不是图像评估，而是一些自定义的隐性规则，例如生成的美团无人车和真实图片不一致，技能命中错误未走指定生图链路
    

- **人工反馈利用效率低：**虽然业务方会提供大量改图意见，但这些经验无法沉淀，容易重复犯错。另一方面，用户反馈也无法及时、高效的反哺Agent，形成奖励策略，辅助Agent正向迭代。
    
- **线上巡检及A/B实验**：线上需求迭代及Agent有较大版本更新时，无法及时定量评估改动的正向影响性；也无法在线上开展A/B实现，实现精准Agent迭代。
    

因此，需要一套体系化、专业化的智能创作评测能力，让 Agent 能够“越用越懂业务”，持续提升评估的准确性，进而实现生图、评估、修正的正向闭环，不断提高采纳率。

#### 1.2 目标

智能创作Agent的评测/观测，不只是“打分”，而是让创作系统变成一个：可观测、可理解、可比较、可自动优化的工程系统。

- **Agent决策和执行过程观测** 通过观测“它是怎么一步步工作的”，来判断框架流程是否合理、冗余。
    
- **线上巡检固定化** 通过自动化线上固定评测集巡检，定时发现线上问题，和告警系统联动，提升系统稳定性及健壮性
    
- **版本更新/实验常态化** 版本更新及实验上线前，需要提交固定评测集的评测结果
    
- **基于人工反馈构建多模态知识库** 构建人工反馈知识库，基于知识库迭代Agent策略，提高评测准确性[基于人工反馈 + 多模态RAG 的自动化Agent评估方案](https://km.sankuai.com/collabpage/2738781405)
    
- **基于线上回流数据，构建Agent训练/自我学习系统** 通过对线上用户原始输入、Agent行为轨迹、用户反馈信息的回流，来进行基于Training / Training-free / self-Evolving的系统迭代。
