## 一、我是谁？

**1024 Agent** 是美团内部的Agent平台，它不是问答机器人，而是**能执行任务的 AI Agent**：理解你的需求之后，会自主拆解步骤、调用内部系统工具、最后交付结果。

- 🧠 能理解你的需求，支持自然语言对话，同时识别图片、语音和文件
    
- 🔧 能调用内部系统（学城、会议室、数据库、大象群等）
    
- 📋 能帮你完成从查询、整理到生成文档的完整工作流
    

> 💡 **简单来说：你说需求，它帮你干活。**

核心价值是让任何人都能低成本创建专属 Agent，无需深度技术背景，通过配置提示词、安装 Skill，即可快速搭建一个面向特定场景的 AI Agent。

### 场景一：角色 Agent

像一个专业的虚拟员工，任何人都可以和它私聊，帮你完成某个特定场景的任务。

|示例场景|说明|
|---|---|
|📊 **数据分析助手**|"帮我查一下本周各城市的 GMV 数据，生成汇总表"|
|📝 **文档写作助手**|帮你起草需求文档、周报、复盘报告等各类文档|
|🔍 **知识检索助手**|从学城、内部系统中快速找到你需要的信息|
|🎯 **面试助手**|模拟面试场景，提供专业的面试辅导和反馈|
|📢 **群内数据播报**|推荐快驴实践：[快驴业务自动化播报工具Skills](https://km.sankuai.com/collabpage/2751240619?kmId=2751240619&linkType=KM)|
>**适用场景：** 有明确专业方向的场景，希望有一个"专家"随时待命，任何人都能直接私聊使用。

### 场景二：团队 Agent

供团队成员统一使用，沉淀团队内的标准化流程和知识资产，让整个团队都能站在巨人肩膀上工作。

|示例场景|说明|
|---|---|
|📢 **团队日报 / 周报**|按模板整理今天的业务日报，发到运营群|
|📚 **知识库问答**|把团队 SOP、最佳实践沉淀进去，新人直接问 Agent|
|🔔 **定时播报提醒**|每天早上 9 点在群里播报今日待办事项|
|📝 **会议纪要整理**|把会议要点整理成结构化纪要，存到学城|
>**核心价值：** 一个机器人 = 一个随叫随到、不请假的虚拟同事。团队共识和 SOP 一旦沉淀进去，所有人都能受益。

### 场景三：客服 Agent

7×24 小时在线回答用户问题，自动处理常见问题，支持多轮对话理解用户意图，大幅降低人工客服压力。

|示例场景|说明|
|---|---|
|💬 **新人答疑机器人**|自动回答新同学的常见问题，减少重复解答|
|🛠️ **技术支持助手**|处理系统使用、配置相关的常见问题|
|📋 **流程引导助手**|引导用户完成申请、审批等标准化流程|
|🔄 **多轮对话理解**|支持上下文追问，理解用户真实意图|

> **适用场景：** 有大量重复性问题需要处理，希望减少人工介入，提升响应速度和一致性。


### 场景四：群助手 Agent

把1024机器人或者业务机器人拉到群中，充当群助手角色，处理群里各种事项，让群聊更高效、更有组织。

> **推荐实践：** 把 Agent 拉进大象群后，整个团队都能直接 @ 它提问、触发任务，一个机器人服务整个团队。

|示例场景|说明|

## 三、1024 vs OpenClaw的区别

> 这是大家最常问的问题：**有了**OpenClaw**，还需要 1024 吗？**[来自用户真实场景实践](https://km.sankuai.com/collabpage/2751370619)

1024和OpenClaw解决的是不同层面的问题，核心差距在**企业级能力**上：

||OpenClaw|1024（企业级 Agent 平台）|
|---|---|---|
|**部署方式**|单沙箱部署|集群化部署，多节点水平扩展，理论支持百万级粒度|
|**高可用**|单点|节点故障自动摘除，服务不中断|
|**并发能力**|个人使用，不支持规模化|支持大规模并发，适合团队 / 业务级使用|
|**数据安全**|个人数据隔离|三层隔离：集群 / Agent / 用户，跨用户访问代码层强制拦截|
|**可观测性**|无|完整 Trace 链路，每步工具调用的入参、出参、耗时全部可查|
|**扩展性**|个人 Skill，难以共享|Agent 级 Skill / MCP / 知识库，团队统一维护、所有人受益|
|**适用场景**|个人长尾工作流|团队标准化流程、业务级自动化、大规模客服|

**一句话总结**

> 🦐 **龙虾**：让你自己每天少加一小时班，是你的「个人数字分身」。
> 
> 🤖 **1024**：让整个团队的运转效率和业务决策质量得到提升，是团队的「虚拟员工」，更支持大规模业务场景使用。

## 五、使用技巧

### 💡 明确输出格式，而不是让它自由发挥

```
❌ "帮我整理一下这次会议的内容"
✅ "帮我整理这次会议纪要，输出三部分：决策结论、待办事项（含负责人）、下次会议议题，存到学城"
```

### 💡 让它帮你"想问题"，而不只是"答问题"

```
反向用法——把它当成一个会提问的审查者：
✅ "我准备明天向老板汇报这个方案，你帮我从老板视角提 5 个最可能被追问的问题"
✅ "这份需求文档有什么遗漏或者逻辑不自洽的地方"
```

### 💡 直接丢文件或截图，省去大段描述

```
1024 支持识别图片、Excel、文档。与其手打一大段背景，不如直接把截图、表格、文档丢给它——信息更准确，你也更省力。
遇到数据分析、页面问题排查、文档改写等场景，优先用附件触发。
```

### 💡 用好记忆：主动让它记住你的偏好，而不是每次重复说

```
很多人不知道可以显式让它记住固定信息：
✅ "记住：我写文档的风格是结论先行，不要长篇铺垫"
✅ "记住：我们团队的日报模板是……以后每次按这个格式"
✅ "记住：提到'北极星指标'默认指 DAU"

项目背景、团队结构、业务术语，说一次就够：
✅ "记住我们项目的基本信息：产品名叫 xxx，目标用户是……核心指标是……"
之后问任何相关问题，不用每次重新解释背景。

记忆多了可能有过时信息，可以定期清理或更新：
✅ "你现在记住了哪些关于我的信息？列出来"
✅ "之前让你记住的日报模板作废了，更新成这个：……"
✅ "清除所有关于 xxx 项目的记忆"
```

### 💡 **链式任务：一次对话完成多步工作流**

```
不用每步单独触发，直接描述整个流程让它串起来执行：
✅ "帮我：1）从学城找到上周的竞品分析文档，2）提取核心结论，3）结合我发给你的这份新数据，生成一页对比总结，存到学城"
```

### 💡 结果不满意？追问比重发更高效

```
❌ （重新发一遍，把所有要求再写一遍）
✅ "第 2 点展开说说"
✅ "语气改得轻松一点"
✅ "按表格格式重新输出"
✅ "结论放到最前面"
```

### 💡 用定时任务，让重复工作自动跑

对于每天/每周固定要做的事，不需要手动触发——直接告诉 1024 设置定时任务。

```
✅ "每天早上 9 点，把昨日销售数据汇总发到【运营日报群】"
✅ "每周一上午 10 点，提醒我填写上周工作总结"
✅ "每天下午 6 点，整理今天的待办完成情况，发给我"
```

设置后无需再操心，1024 会按时执行。适合数据播报、定时提醒、周期性报告等场景。如需调整或取消，直接说"停掉每天早上的数据播报"即可。

### 💡 多轮对话积累上下文，处理长周期任务

```
在同一个会话里持续补充信息，而不是每次都从头说：
第一轮："我在做一个用户调研，目标是……"
第二轮："这是今天收集到的 20 条反馈，帮我归类"
第三轮："结合之前的目标，现在写结论"
```

### 💡 结合 Skill 组合使用，打通数据到文档的完整链路

```
安装数据类 Skill 后，可以把"取数 → 分析 → 写文档 → 存学城"一气呵成：
✅ "查一下本周外卖各城市 GMV，和上周对比，生成简报存到学城，然后发给我"
```

### 💡 把它拉进团队群，释放团队价值

将 1024 机器人拉入大象群，整个团队都能直接 @ 它提问、触发任务。配合定时播报、群内答疑，一个机器人能服务整个团队。
