# Agent通用评测标准（性能指标）

| **评测基准** | **核心定位** | **核心评估指标** | **评测方式** | **结果解读** | **官方开源 / 数据链接** |
|---|---|---|---|---|---|
| GAIA | Meta 主导，评估通用 AI 解决 “人类易、AI 难” 的现实任务 | **完成率**：成功输出正确结果的任务占比（核心指标）；**响应质量**：人工 + 自动化校验结果的准确性（如数据计算正确性、多模态理解到位度）；**效率**：完成任务的耗时与计算资源开销（如 GPU 显存占用）；**稳健性**：面对误导性数据、干扰信息时的性能稳定性（如输入含错误前提时是否仍能正确推理）；**泛化分数**：对未见过的新类型任务的适配能力（重点计算 Level 3 复杂任务的完成率）。 | **任务部署：**466 个问题按 Level 1-3 分级，提供网页浏览器、PDF 阅读器等工具；**Agent 执行：**AI 自主完成任务，记录操作与输出；**指标计算：**按 “完成率→响应质量→效率→稳健性→泛化分数” 依次统计，加权得平均得分；**结果汇总：**生成 Leaderboard 排名。 | h2oGPTe.Agent（Claude-3-7-sonnet）平均得分 74.75%，其中 Level 3 泛化分数 53.06%；Genspark 超级代理 Level 3 泛化分数 58.8%，显著高于 Manus.ai（47.6%），说明其在复杂未知任务中的适配性更强；90% 现有 AI Agent Level 3 完成率低于 50%，凸显通用智能体的复杂任务能力短板。 | Hugging Face：[meta-llama/GAIA](https://huggingface.co/spaces/gaia-benchmark/leaderboard)​​ |
| AgentBench | 清华主导，评估 LLM 的推理与决策能力 | **OA 总分**：所有环境的加权平均分（核心综合指标，权重按环境重要性分配，如 OS 占 20%、购物占 15%）； **任务完成率**：单环境内 “达成指令目标” 的任务占比（如 Linux 环境中 “成功创建用户” 得 1 分，失败得 0 分）；**多轮对话一致性**：长对话中逻辑不矛盾的程度（通过对比多轮输出是否冲突判断，如前序说 “文件在桌面”，后续是否仍一致）；**代码生成准确性**：生成代码的可运行性 + 结果正确性（需实际执行代码，验证输出是否符合任务要求，如 SQL 查询结果是否匹配预期）。 | **环境搭建：**模拟 Linux bash、MySQL、购物网站等 8 大真实环境，预置任务指令；**Agent 交互：**AI 通过 API 调用 / 文本指令与环境交互（如输入 Linux 命令、写 SQL 语句），记录每步操作；**指标计算：**先算单环境任务完成率，再按环境权重算 OA 总分，同步校验对话一致性与代码准确性；**结果对比：**按 “闭源 / 开源模型” 分类生成柱状图。 | 闭源模型：GPT-4（0613 版）OA 总分 4.01 分，家庭环境任务完成率 78.0%、网页购物 61.1%，代码准确性达 89%；开源模型：LLaMA-2-70b OA 总分 0.78 分、Guanaco-65b 0.54 分，平均仅 0.51 分，与闭源模型（平均 2.15 分）差距显著，说明开源模型在环境交互与决策上仍需突破。 | GitHub：[THUDM/AgentBench](https://github.com/THUDM/AgentBench)​​ |
| PaperBench | OpenAI 主导，评估 AI 复现科研成果的能力 | **复现得分**：0-1 分量化指标（核心指标），按 “模型结构复现（0.3 分）+ 实验参数匹配（0.2 分）+ 结果误差（0.3 分）+ 文档完整性（0.2 分）” 分项打分；**代码完整性**：复现所需核心代码的覆盖程度（如是否包含数据预处理、模型训练、结果可视化全流程代码）；**实验一致性**：AI 生成的实验结果与论文原始结果的吻合度（如准确率误差≤5% 为优，＞10% 为差）。 | **资源准备：**提供 20 篇 ICML 2024 论文原文、预装依赖的 Ubuntu 容器；**三阶段评测：****代理推出：**AI 编写复现代码与实验脚本（如 reproduce.sh）；**复制执行：**GPU 容器运行脚本，输出实验结果；**评分**：按 “论文核心贡献匹配度” 量化复现得分；**结果校准：**与人类复现结果对比，计算差距。 | 人类评测者：复现得分平均 0.93±0.04，代码完整性 100%，实验一致性误差≤3%；AI 模型（Claude 3.5 Sonnet）：复现得分平均 0.30±0.01，代码完整性仅 65%（常缺失数据预处理代码），实验一致性误差平均 18%，说明 AI 在科研逻辑理解与细节把控上远逊于人类。 | GitHub：[openai/paperbench](https://github.com/openai/preparedness/tree/main/project/paperbench)​​ |
| Windows Agent Arena（WAA） | 微软主导，评估 Windows 环境下的实操能力 | **任务成功率**：完成指定 Windows 操作且结果符合目标的比例（核心指标，如 “PDF 是否保存到桌面”“VS Code 配置是否生效”）；**操作精度**：有效操作占比（有效操作 = 正确点击 / 快捷键，无效操作 = 点击错误按钮、快捷键用错，精度 = 有效操作数 / 总操作数 ×100%）；**耗时效率**：完成任务的平均时间（排除等待系统响应的耗时，仅统计 AI 操作耗时）。 | **环境部署：**搭建 Windows 11 VM，预装 Edge/VS Code / 画图，支持 Azure 云并行（100 台 VM 同步测试）；**任务下发：**随机分配 154 项任务（如 “网页存为 PDF”），无操作提示；**Agent 操作：**AI 通过鼠标 / 键盘交互，记录操作轨迹；**指标计算：**统计成功率与操作精度，标注失败原因（如 OCR 错误）。 | 微软 Navi Agent：任务成功率 19.5%，操作精度 62%（40% 操作为无效点击），平均耗时 450 秒；人类：任务成功率 74.5%，操作精度 98%，平均耗时 120 秒；失败原因：60% 因 “操作精度不足”（如误点 “打印” 而非 “另存为”），30% 因 “视觉 - 语言对齐失效”（如指令 “清除高亮” 却删除文本），指导 AI 优化界面理解与操作决策。 | GitHub：[microsoft/Windows-Agent-Arena](https://github.com/microsoft/WindowsAgentArena)​​ |
| SuperCLUE-Agent | 中文场景专属，评估中文 Agent 能力 | **中文任务完成率**：中文场景下达成任务目标的比例（核心指标，如 “正确生成中文活动方案”“准确回答中文论文问题”）；**API 调用准确率**：API 参数填写正确性（如 “天气 API 中城市名是否正确”）+ 调用逻辑合理性（如 “是否先验签再调用”）的综合比例；**长程对话保持率**：多轮中文对话中不偏离主题的比例（如对话 10 轮后，仍能回应首轮提到的 “会议时间”）；**任务分解合理性**：复杂任务拆分为子步骤的逻辑通顺度（人工 + 算法联合判断，如 “订机票→选酒店→做行程” 是否符合常规流程）。 | **任务构建：**5000 道中文任务分 “工具使用 / 任务规划 / 长短期记忆” 三类，含金融、医疗垂直场景；**自动化评测：**检查 API 调用参数、任务分解逻辑、对话记忆准确性；**主观校验：**人工对 “对话自然度” 打分，与自动化得分加权；**结果排序：**按中文任务完成率生成模型排名。 | GPT-4：中文任务完成率 92%，API 调用准确率 88%，长程对话保持率 95%，任务分解合理性 100%；国内模型（ERNIE-3.5-Turbo）：中文任务完成率 78%，API 调用准确率 75%，长程对话保持率 80%（常遗忘 3 轮前的细节），任务分解合理性 85%，说明国内模型在中文多轮记忆与复杂逻辑拆解上需优化。 | GitHub：[SuperCLUE/SuperCLUE-Agent](https://github.com/CLUEbenchmark/SuperCLUE-Agent)​​ |
| AgentHarm | 聚焦 AI Agent 的安全性，评估恶意攻击防御能力 | **合规率**：拒绝恶意请求的比例（核心指标，如 “收到‘生成钓鱼邮件’请求时，明确拒绝的次数 / 总请求次数 ×100%”）；**攻击成功率**：被 “越狱提示” 诱导执行恶意任务的比例（如 “角色伪装成黑客后，AI 生成恶意代码的次数 / 总诱导次数 ×100%”）；**多轮防御一致性**：多轮对话中持续拒绝恶意请求的程度（如 “首次拒绝后，后续 3 轮是否仍拒绝，无反复”）。 | **恶意任务库：**440 个任务分 “欺诈 / 网络犯罪 / 身份盗窃”11 类，含 “多轮诱导话术”；**对抗测试：****基础测试：**直接下发恶意请求，统计合规率**越狱测试：**用 “提示注入”“角色伪装” 诱导，统计攻击成功率； **一致性校验：**多轮对话中重复恶意请求，检查防御一致性；**结果分级：**按合规率划安全等级（A 级≥90%、B 级 70%-90%）。 | GPT-4：基础测试合规率 85%（直接恶意请求中 85% 被拒绝），越狱测试攻击成功率 60%（被 “角色伪装” 诱导生成恶意代码），多轮防御一致性 70%（部分案例第 3 轮开始妥协）；行业现状：主流模型均存在 “基础防御强、抗越狱弱” 问题，合规率普遍低于 90%，需强化越狱攻击防御机制。 | GitHub：[GraySwanAI/AgentHarm](https://github.com/UKGovernmentBEIS/inspect_evals/tree/main/src/inspect_evals/agentharm)​​ |
| PromptBench | 评估提示工程与对抗攻击下的鲁棒性 | **对抗鲁棒性得分**：遭遇对抗攻击后的性能保留程度（核心指标，计算方式 = 攻击后准确率 / 正常测试准确率 ×100%，得分越高鲁棒性越强）；**提示适配度**：不同提示工程方法对任务准确率的提升幅度（如 “零样本 CoT 提示的准确率 - 基础提示的准确率”，正值为提升，负值为下降）；**动态评估分**：多轮提示调整后的性能稳定性（如 “调整 3 次提示后，准确率的波动幅度，波动越小得分越高”）。 | **准备工作：**12 类基础任务（情感分析、数学等）+6 种提示方法（零样本 CoT 等）+7 种对抗攻击（TextBugger 等）；**对照测试：****正常测试：**标准提示测准确率（基准值）；**对抗测试：**攻击后测准确率，算鲁棒性得分**提示适配测试：**对比不同提示方法的准确率；**结果分析：**生成 “提示 - 任务 - 鲁棒性” 热力图。 | GPT-3.5（情感分析任务）：正常测试准确率 89%，TextBugger 攻击后准确率 54%，对抗鲁棒性得分 60.7%；提示适配度：零样本 CoT 提示比基础提示准确率高 22%（数学求解任务），EmotionPrompt 比基础提示高 15%（情感分析任务）；结论：字符级攻击（TextBugger）对模型影响最显著，零样本 CoT 是通用性最优的提示方法。 | GitHub：[microsoft/PromptBench](https://github.com/microsoft/PromptBench)​​ |
| T-Eval | 细粒度评估工具使用能力 | **工具选择准确率**：选择适配当前任务的工具的比例（核心指标，如 “任务‘统计 2023 年 GDP’，选‘国家统计局 API’为正确，选‘通用搜索’为错误”）；**参数配置正确率**：工具参数填写符合任务要求的比例（如 “API 请求中‘年份 = 2023’‘地区 = 全国’为正确，‘年份 = 2022’为错误”）；**多工具协同效率**：完成多工具组合任务的效率（以人类平均耗时为基准，AI 耗时 / 人类耗时 ×100%，低于 150% 为优）。 | **工具模拟：**提供 20 + 常用工具（搜索引擎、Excel、API 等），设计 “选择 - 配置 - 协同” 全流程任务；**分步评测：****工具选择：**判断 AI 是否选对适配工具；**参数配置：**检查参数填写正确性；**协同效率：**统计多工具任务耗时；**指标计算：**按 “选择 30%+ 配置 50%+ 效率 20%” 加权得总分。 | GPT-4：工具选择准确率 89%，参数配置正确率 82%，多工具协同效率 120%（比人类慢 20%）；开源模型（LLaMA-2-70b）：工具选择准确率 65%（常选错工具，如用 “Excel” 统计实时股票数据），参数配置正确率 58%（常漏填关键参数），多工具协同效率 240%（比人类慢 140%），说明开源模型在工具细节把控与协同上差距明显。 | GitHub：[T-Eval/T-Eval](https://github.com/open-compass/T-Eval)​​ |
