## 一、组件概览

### 功能定位

步骤条（Steps）由 2 个及以上节点串联而成，用于引导用户分步完成任务，或仅展示任务进度及状态。

### 适用场景

1. 多步骤表单填写流程（如注册、下单、申请）
    
2. 任务进度展示（如订单状态、审批流程）
    
3. 引导式操作流程（含操作按钮的纵向步骤条）
    
4. 紧凑型进度指示（页面顶部横向进度条）
    

### 组件变体

|变体名称|视觉特征|使用场景|
|---|---|---|
|横向步骤条（大尺寸）|图标 24pt，标题 16pt，水平排列|步骤数 ≤ 4，空间充裕|
|横向步骤条（小尺寸）|图标 20pt，标题 14pt，水平排列|步骤数较多或空间有限|
|纵向步骤条（大尺寸）|图标 24pt，标题 18pt，垂直排列，可含引导按钮|步骤内容较多，需展示辅助信息|
|纵向步骤条（小尺寸）|图标 20pt，标题 14pt，垂直排列|步骤内容简洁的纵向场景|
|点状步骤条|圆点替代数字图标，垂直排列|仅展示进度状态，不强调步骤序号|
|紧凑步骤条|整体高度 32pt，水平紧凑排列|页面顶部进度指示，空间极度有限|

## 二、设计规范

### 2.1 布局结构

**主布局：** flex 布局，横向变体为 `flex-direction: row`，纵向变体为 `flex-direction: column`

**层级结构：**

```
步骤条容器（Steps）
├── 步骤节点（Step Item）× N
│   ├── 图标区（Step Icon）          // 圆形图标，含数字/勾/×
│   ├── 标题（Step Title）           // 步骤名称
│   └── 辅助信息（Step Description） // 可选，补充说明
└── 进度线条（Progress Line）× (N-1) // 连接相邻步骤节点
```

**纵向步骤条额外层级：**

```
步骤节点（Step Item）
├── 左侧轴线区
│   ├── 图标（Step Icon）
│   └── 进度线条（Progress Line）   // 垂直，1pt 宽
└── 右侧内容区
    ├── 标题（Step Title）
    ├── 辅助信息（Step Description）
    └── 引导按钮（Action Button）   // 可选，仅进行中步骤显示
```

**子元素排列：** 横向变体各步骤节点等宽排列，图标居中对齐；纵向变体图标左对齐，内容区靠右。

**关键布局说明：**

1. 横向步骤条中，进度线条位于两个步骤节点图标之间，水平居中于图标中心点。
    
2. 纵向步骤条中，进度线条垂直居中于图标，从图标底部延伸至下一个图标顶部。
    
3. 点状步骤条的进行中节点使用双圆结构（外圆 14pt + 内圆 12pt），视觉上形成环形效果。
    
4. 紧凑步骤条中，步骤节点与分隔符水平排列，整体高度固定为 32pt，不展示辅助信息。
5.
