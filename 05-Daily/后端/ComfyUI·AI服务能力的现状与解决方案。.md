### 当前遇到的问题

#### ComfySpark的升级迭代问题

一个ComfyUI的版本对应一个基座，也就是限定了可兼容的组件版本范围

想要使用最新的功能、最新的组件来提升设计效率和生图效果就必须升级基座，可是目前ComfySpark处于仅维护状态

#### 部署发布的效率问题

依赖TPP发布，哪怕做一个简单的工作流微调，部署到线上需要1-2小时

#### 开发工作流的效率问题

开发机上的WebUI，虽然支持多人独立编辑，但是无法并发运行（只有一张显卡）

#### 服务的并发能力

tpp的部署模式，一个节点需要一张L20显卡，显卡资源有限，影响了并发能力

目前是通过排队机制解决

**淘天AI设计----AI服务器能力诉求。**

**01、调用外部的API能力。**

支持通过 token、AK/SK 等鉴权方式在自定义节点中调用 SaaS 接口

阿里内网的ideaLab的多模态模型、生图模型接口

阿里内网的其他API调用，比如抠图、扩图能力

能够像目前Comfyui最新版的API节点集合一样调用各个外部API接口，收费和鉴权方式走Comfyui的渠道

尤其是BFL Flux API的调用

要保证可以这种调用方式可以大量并发

能通过git命令拉取存储于阿里内网git仓库的代码

**02、自有部署的API能力**

支持通过 HTTP API 接收参数调用工作流生成图像（传入业务参数）

传入工作流JSON（包含提示词、模型调用、图片URL）

能获取到队列信息 -> 当前用户任务在该实例队列的第几个

Comfyui基础的API调用都需要支持

支持将单一工作流部署为单容器，并将该容器的调用部署为API形式

API可传入文案、图片URL、鉴权信息

API 层支持鉴权、异步调用、结果回调，可灵活接入淘天业务系统

**03、后期维护**

能够随时隔离测试新的节点（自研或社区的节点）和python环境，不影响运行中的Comfyui实例，测试完毕后可以方便地（一键）发布到指定的其他Comfyui实例

能够持续与Comfyui同步更新
