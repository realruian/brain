### 翻车的部分：刷新时图片 flicker

**现象**：Col 3 图首次打开 OK，**刷新时瞬间变扁**再回到 3:2。

**尝试了 6 轮修复，全部失败**：
1. `.img-wrap` 加 `aspect-ratio` + img `height: 100%`
2. `.img-wrap` 加 `width: 100%`
3. img 加 `position: absolute; inset: 0`
4. 换 padding-bottom hack（用户明确拒绝老派方案）
5. aspect-ratio 从 parent 挪到 img 本身
6. img 加 HTML `width="1200" height="800"` + CSS `height: auto`
7. parent 加 `contain: size layout`
8. shader 未 ready 前 img `opacity: 0`

Console 诊断证明稳定态 `.img-wrap` ratio=1.500 一直正确，变形只在刷新一帧内。我在**没有"那一帧"的精确数据**情况下连续猜了 6 次都没中。

**最后回滚** `git checkout ea96ee9 -- styles.css ...` 放弃所有 flicker 修复，回到简单状态。问题暂留。

### 晚上最大的教训

**没有精确诊断数据时，不要连续改同一个文件 6 次。**

每次改之前应该先：
- 用 `ResizeObserver` / `Performance` 录 reload
- DevTools Elements → Computed panel 截图"问题瞬间"
- 拿到真实状态再改，否则就是在概率分布里随机抽样

盲改 6 次 = 浪费 1 小时 + 搞乱 CSS + 最后还是要回滚。

### 正确姿势（下次要做到）

1. 看到问题 → **先抓数据**（Console 监听 / DevTools 断点 / Performance 录制）
2. 看清状态 → **再定假设**
3. 验证假设 → 再改 1 次
4. 失败 → 回到步骤 1 重新取数据，**不要**凭感觉改第 2 次

今天的错是跳过 1–2 直接进 3，还重复了 6 次。
