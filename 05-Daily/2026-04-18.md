### 翻车的部分：刷新时图片 flicker

**现象**：Col 3 图首次打开 OK，**刷新时瞬间变扁**再回到 3:2。

**尝试了 6 轮修复，全部失败**：
1. `.img-wrap` 加 `aspect-ratio` + img `height: 100%`
2. `.img-wrap` 加 `width: 100%`
3. img 加 `position: absolute; inset: 0`
4. 换 padding-bottom hack（用户明确拒绝老派方案）
5. aspect-ratio 从 parent 挪到 img 本身
6. img 加 HTML `width="1200" height="800"` + CSS `height: auto`
7. parent 加 `contain: size layout`
8. shader 未 ready 前 img `opacity: 0`

Console 诊断证明稳定态 `.img-wrap` ratio=1.500 一直正确，变形只在刷新一帧内。我在**没有"那一帧"的精确数据**情况下连续猜了 6 次都没中。

**最后回滚** `git checkout ea96ee9 -- styles.css ...` 放弃所有 flicker 修复，回到简单状态。问题暂留。

### 晚上最大的教训

**没有精确诊断数据时，不要连续改同一个文件 6 次。**

每次改之前应该先：
- 用 `ResizeObserver` / `Performance` 录 reload
- DevTools Elements → Computed panel 截图"问题瞬间"
- 拿到真实状态再改，否则就是在概率分布里随机抽样

盲改 6 次 = 浪费 1 小时 + 搞乱 CSS + 最后还是要回滚。

### 正确姿势（下次要做到）

1. 看到问题 → **先抓数据**（Console 监听 / DevTools 断点 / Performance 录制）
2. 看清状态 → **再定假设**
3. 验证假设 → 再改 1 次
4. 失败 → 回到步骤 1 重新取数据，**不要**凭感觉改第 2 次

今天的错是跳过 1–2 直接进 3，还重复了 6 次。

## 凌晨翻盘：flicker 真凶找到 + Writing section 上线

睡前不甘心，又开了一会，结果一晚上把昨天遗留全部解决，外加做完一个新功能。

### 1️⃣ flicker 真凶（commit 1b861ab）

**关键洞察来自一句话**：用户告诉我"本地没事，线上才闪"。这一句直接把 6 次失败的 layout 修复方向全否了——本地和线上**唯一变量是网络 RTT**。

按昨晚自己写的"先抓数据再改"的规矩，用 Chrome MCP 在线上跑全套：

- `performance.getEntriesByType('layout-shift')` → **空数组**（box 几何零变化）
- ResizeObserver 在 src 重置时 → **零回调**（img/wrap layout box 不变）
- `drawImage + getImageData` 采样 img 像素 → 彩色（RGB diff 16.8）
- 截屏看屏幕 → 黑白 halftone
- 结论：flicker **不是 layout 问题，是像素层面的视觉切换**

读 `js/shaders.js` 找到真凶：
```js
async function initShader(wrap) {
  ...
  wrap.dataset.shaderInit = '1';   // ← line 24，太早！
  ...
  await new Promise(r => { ... });  // 等图片加载 150ms
  ...
  root.render(...);                 // React + WebGL 200-400ms
}
```

`data-shader-init = '1'` 在 React mount + WebGL 编译之前就设了。CSS `:not([data-shader-init="1"])::after` 米色点阵占位**提前 200-600ms 让位**，露出 img 原图 → halftone 切换的视觉窗口。

修复：标志移到 `root.render` + double RAF 之后；防重复 init 用内部 `_shaderStarted` flag 解耦。**改了一行的位置 + 加 4 行 RAF + 加 catch 兜底**，问题彻底消失，达到 dany.works 同等水准。

### 3️⃣ 今夜最大教训：标志位的双重职责必须拆分

shader-init 这一个 flag 同时管两件事：① 防 shader 重复 init；② 触发 CSS 占位让位。**合二为一就两难**：早设防住并发但 UI 闪、晚设 UI 不闪但可能并发重入。

正确解法：拆成两个独立 flag——
- 内部 flag（早设）→ 防重复
- DOM 属性（晚设）→ 触发 UI 状态切换

已写入 `~/.claude/CLAUDE.md` 作为通用规则，下次任何项目遇到"标志位同时干两件事"立刻警觉。
