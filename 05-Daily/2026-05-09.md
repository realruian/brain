### 教训（可能值得写进 CLAUDE.md）

1. **基线规则要考虑尺寸边界**：`:where(...):focus-visible` 一刀切给所有 input/textarea 加 outline，小元素合适但大尺寸 textarea 灾难。任何"全局 base 规则"在写之前要列举"最大尺寸 instance"看视觉。

2. **useState 缓存让 exit 动画继续渲染时，所有 hooks 必须放 early return 之前**：React Rules of Hooks 严格要求每次 render hook 序列一致。`if (!data) return null` 能用前提是放在所有 hooks 之后，不能放中间。

3. **dogfood > typecheck/lint**：类型检查不抓 React runtime invariant（hooks 规则、effect 时序、AnimatePresence 配合 unmount）。任何涉及 hooks 模式的改动都必须浏览器实测。

4. **加 motion 的策略**：先 1-2 个核心 dogfood 看反馈再加下一批，不要一次堆 10 个。Material 3 / Polaris 都明确"add motion deliberately"。

5. **改 CTA 文案前要核对这一步真正做什么**：§10.2 规则"具体动词 + 对象"不能机械应用，"对象"要准确。「设置岗位」误导用户以为要填岗位信息，实际是配置面试。

6. **eslint-disable 注释必须带原因 + 修订链接**：未来读代码人才能判断是否还需要。
