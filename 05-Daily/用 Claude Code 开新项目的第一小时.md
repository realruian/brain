# 先搭 Harness 再写代码：Claude Code Harness 速查

> Harness 指的是你围绕 Claude Code 搭起来的工作台 —— 项目里的 CLAUDE.md、知识库里的项目笔记、Hooks、Skills。搭好之后每次对话不用重新铺垫上下文。

核心就四句话。

---

## 开新项目的第一小时（无论从零还是接入既有）

**一句话记住**：**先建 harness，再写代码**。跳过 harness 的那几分钟，以后会用几小时偿还。

### 三步仪式

**① 建项目 CLAUDE.md**（项目根目录，控制在 100 行内）

对 Claude 说：

```
读一下代码结构（或者：我要做一个 XX 项目），给我生成 CLAUDE.md：
技术栈 / 本地启动命令 / 验证手段 / 不要动的文件 / Git 纪律（push 前 pull、不要 force 之类）
草稿就好，我审查精简
```

**② 建项目笔记**（放在你的知识库里，比如 Obsidian）

```
在我的知识库项目目录下建一个 xxx.md，包含：
业务背景 / 关键决策 / 已知坑点 / 进展记录
空骨架即可
```

**③ 把项目登记到全局 CLAUDE.md 的"当前项目"**

这样跨项目对话时 Claude 也知道你在做什么。

```
在全局 CLAUDE.md 的"当前项目"里加一行 xxx
```

---

## 日常开发

**每次开新会话就一句话恢复上下文**：

```
读项目 CLAUDE.md 和项目笔记，然后看 git status + git log -5，告诉我从哪里继续
```

**写代码时**：

- 超过 3 步或涉及架构决策 → 先进 **Plan Mode**（按 `Shift+Tab`）让 Claude 出方案
- 简单改动 → 直接 Auto-accept edits

**犯错之后，对 Claude 说这句**：

```
把刚才这个教训加到项目的 CLAUDE.md，防止以后再犯
```

这句话是整套流程里最重要的一句 —— 错一次就变规则，下次不再犯。CLAUDE.md 就是这样一圈圈长出来的。

---

## 收工（3 分钟）

```
看一下今天的 git log，帮我写今日总结到每日笔记里（文件名用今天日期）
顺便更新项目笔记的进展记录
```

---

## 如果只记一个要点

> **每次犯同一个错两次，就对 Claude 说"加到 CLAUDE.md"。**

其他进阶玩法（Hooks / Skills / Plan Agent / 模块化 rules）等你觉得「CLAUDE.md 不够用了」再加 —— 90% 的项目这一步永远不会到。

先把三步仪式跑通，比任何高级配置都更能让 Claude 稳定产出。
