## 教训

- **不要急着 hypothesize 根因**。看到 bug 先用 AskUserQuestion 确认场景(在哪个 URL 触发 / 看到什么文字 / 时长),再写代码。多次跳跃猜测白白浪费时间。
- **看到"文案几乎一样"就警觉**。MessageStream 空态文案 vs jobs 主页文案差 3 个字,视觉无法区分,以为是"路由跳错"其实是"empty state 误触发"。debug 先确认看到的标题精确文字。
- **能 grep 不要让用户贴 console**。用户火了是合理反馈,自己 grep / read code 能解决的事不要拽用户。
- **router.replace 适合"放弃当前页"语义**。用户从 /report 报告页"返回主页",是放弃当前页不留 history。push 在历史栈复杂时 Next cache 行为不可预期,replace 更稳。
