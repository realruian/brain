# Task A · 数据表改造方案（Schema Overhaul Proposal）

## 现状一句话

`templates` 表是一张**平表**，`category` 是裸字符串，没有业务线维度，没有模型登记，没有使用统计；`psd_layers` 结构基本够用，不动。

## 改造目标

从淘天搬**四个维度**进来，但保持"模板为中心"的现有心智不变：

|维度|淘天表|现状|改造动作|
|---|---|---|---|
|**业务线**|`ai_img_biz` + `ai_img_user_biz_rel`|无|新增 `businesses` + `user_biz_rel`（先建空表，保留入口）|
|**场景分类**|`ai_img_template_category`|`templates.category` 裸字符串|抽出 `template_categories` 字典表；`templates.category` 改 `category_id` FK|
|**尺寸白名单**|`ai_img_img_size`（挂在 category 上）|无|新增 `category_sizes`（`category_id` × `width` × `height`）|
|**生成能力**|`ai_img_generation_model` + `generation_type` 枚举|无|新增 `generation_models` 登记表；`templates` 加 `generation_type` 字段|
|**使用次数**|`ai_img_common_usage_count`（target_type/target_id）|无|新增 `usage_counts`（多态计数表）|

## 改造后的表结构（DDL 草案）

### 1. `businesses` — 业务线字典（新）

```
CREATE TABLE IF NOT EXISTS businesses (  id            TEXT PRIMARY KEY,           -- e.g. 'waimai-hero', 'brand-ads'  name          TEXT NOT NULL,              -- '外卖会场'、'品牌广告'  order_score   INTEGER NOT NULL DEFAULT 0,  created_at    TIMESTAMPTZ NOT NULL DEFAULT NOW());
```

> 当前没有鉴权，**先建表不建关系**，`user_biz_rel` 留到 S2 接 SSO 时再加。

### 2. `template_categories` — 场景字典（新，取代裸 category）

```
CREATE TABLE IF NOT EXISTS template_categories (  id            TEXT PRIMARY KEY,           -- 'hero-banner', 'entrance-poster'  name          TEXT NOT NULL,              -- '会场头图'、'站内资源位'  business_id   TEXT REFERENCES businesses(id),  order_score   INTEGER NOT NULL DEFAULT 0,  created_at    TIMESTAMPTZ NOT NULL DEFAULT NOW());
```

### 3. `category_sizes` — 尺寸白名单（新）

```
CREATE TABLE IF NOT EXISTS category_sizes (  id            SERIAL PRIMARY KEY,  category_id   TEXT NOT NULL REFERENCES template_categories(id) ON DELETE CASCADE,  width         INTEGER NOT NULL,  height        INTEGER NOT NULL,  label         TEXT,                       -- '750x1334 手淘首焦'  UNIQUE (category_id, width, height));
```

> 用处：上传模板 / 选尺寸时强约束 + 首页按尺寸过滤。

### 4. `generation_models` — 模型登记（新）

```
CREATE TABLE IF NOT EXISTS generation_models (  id            TEXT PRIMARY KEY,           -- 'volcano-seedream-3', 'gpt-image-1'  name          TEXT NOT NULL,              -- 'Seedream 3.0'  capability    TEXT NOT NULL,              -- 'text2img' | 'img2img' | 'inpaint' | 'outpaint' | 'upscale' | 'matting'  invoke_info   JSONB NOT NULL DEFAULT '{}',-- { endpoint, apiKey envKey, timeout, gatewayType }  description   TEXT,  enabled       BOOLEAN NOT NULL DEFAULT true,  created_at    TIMESTAMPTZ NOT NULL DEFAULT NOW());
```

### 5. `usage_counts` — 多态计数（新）

```
CREATE TABLE IF NOT EXISTS usage_counts (  id            SERIAL PRIMARY KEY,  target_type   TEXT NOT NULL,              -- 'template' | 'category' | 'model'  target_id     TEXT NOT NULL,  count_type    TEXT NOT NULL DEFAULT 'use',-- 'use' | 'export' | 'view'  count         BIGINT NOT NULL DEFAULT 0,  updated_at    TIMESTAMPTZ NOT NULL DEFAULT NOW(),  UNIQUE (target_type, target_id, count_type));CREATE INDEX IF NOT EXISTS idx_usage_counts_rank  ON usage_counts (target_type, count_type, count DESC);
```

> **决策**：不在 `templates` 表里加 `usage_count` 列——计数是热点写入，混在主表里会让读请求跟写频繁竞争。独立表后续还能扩 view/export/favorite。

### 6. `templates` — 改造现有表

```
-- 新增列（全部幂等，沿用你现在的 DO $$ IF NOT EXISTS 风格）ALTER TABLE templates ADD COLUMN business_id       TEXT REFERENCES businesses(id);ALTER TABLE templates ADD COLUMN category_id       TEXT REFERENCES template_categories(id);ALTER TABLE templates ADD COLUMN generation_type   TEXT;   -- 'static' (纯模板套字/换图，现状)   -- 'text2img-assisted' (文案靠 LLM 生成)   -- 'img2img-assisted' (主图靠模型换背景/扩展)ALTER TABLE templates ADD COLUMN generation_model_id TEXT REFERENCES generation_models(id);ALTER TABLE templates ADD COLUMN publish_status    TEXT NOT NULL DEFAULT 'published';   -- 'draft' | 'published' | 'archived'ALTER TABLE templates ADD COLUMN extra_info        JSONB NOT NULL DEFAULT '{}';   -- 预留：prompt template、素材链接、运营标签
-- category 保留为兼容列一段时间，迁移完数据后 DROP
```

> **保留字段**：`category`（string）保留一个 sprint 过渡期，读路径双写，确认 `category_id` 稳定后再删。

### 7. `psd_layers` — **不动**

图层表职责清晰，跟业务/能力解耦，不接入这一波改造。

## 迁移步骤（给 Cursor 的执行顺序）

1. **新表建 5 张**（businesses / template_categories / category_sizes / generation_models / usage_counts）——全走幂等 DDL，沿用 `init-db` 路由。
2. **`templates` 加 6 列**——同样幂等 ALTER。
3. **数据回填脚本**（scripts/migrate-category-to-id.ts）：
    - 从现有 `templates.category` distinct 出来一批，inserrt 进 `template_categories`（id = slugify）
    - UPDATE `templates SET category_id = ...` 按 category 文本匹配
4. **改 lib/templates-db.ts**：
    - `rowToTemplate` / `rowToPsdLayer` 加新字段映射
    - `getAllTemplates` JOIN `template_categories` 带出 `categoryName`、`businessId`
    - 新增 `listBusinesses()` / `listCategories(bizId?)` / `listSizes(categoryId)` / `getModel(id)` / `bumpUsage(target_type, target_id)`
5. **`updatePsdLayer` 的 N-SQL 老问题顺手修掉**：
    - 改成动态构造一条 `UPDATE ... SET col1=$1, col2=$2 ...`；用 `sql.unsafe` 或手拼白名单列。
    - **注意**：`@neondatabase/serverless` 的 tagged template 不支持动态列，需要用 `sql.query(text, values)`。
6. **types/template.ts 同步改**：
    - `Template.category: string` → `category: { id: string; name: string }`（JOIN 结果）
    - 新字段 `businessId` / `generationType` / `generationModelId` / `publishStatus` / `extraInfo`
7. **`/api/admin/init-db` 改完**跑一次，再跑 scripts/migrate-category-to-id.ts 一次。

## 风险 & 取舍

- **风险 1：`category` 裸字符串→FK 迁移**。如果历史数据 category 拼写不一致（"会场头图" vs "会场 头图"），slugify 要带 trim+nfkc 归一。建议先跑个 dry-run 脚本 print distinct 出来人工 review 一眼。
- **风险 2：Neon serverless 的 tagged template 不支持动态列**。`updatePsdLayer` 的重构需要切到 `sql.query()` 接口，白名单列名防注入。这点**在本轮先不做**，免得改动面太大。
- **取舍 1：`usage_counts` 独立表 vs `templates.usage_count` 列**——选独立表，理由见上。
- **取舍 2：`user_biz_rel` 现在不做**——现状无认证，留空表+迁移脚位就行。
- **取舍 3：`publish_status` 默认 `'published'`**——因为现存模板都在用，不加 draft 状态就默认是可见的。
